## 3. 问题根因分析

### 3.1 原始评测方式的问题

**训练时的格式**（data_processor.py）：
```python
messages = [
    {"role": "user", "content": question},  # 直接使用原始问题
    {"role": "assistant", "content": answer}
]
```

**原始评测时的格式**（scripts/evaluate.py）：
```python
messages = [
    {"role": "user", "content": question}  # 与训练一致
]
# 但实际使用了不同的生成参数
```

**问题**：
1. 虽然格式与训练一致，但可能存在其他配置差异
2. 批量推理 vs 单条推理的差异
3. 生成参数可能不同

### 3.2 exp0评测模式的关键改进

1. **使用详细的Prompt模板**：
```
你是一个地理空间推理专家。请根据问题给出准确、简洁的答案。

问题: {question}

请直接给出答案，不需要解释过程。答案格式要求：
- 方向问题：直接说明方向，如"东南方向"
- 距离问题：给出具体数值，如"约1200公里"
...
```

2. **单条推理模式**（batch_size=1）

3. **一致的生成参数**：
   - max_new_tokens: 256
   - temperature: 0.1
   - top_p: 0.9
   - do_sample: true

## 4. 关键发现

### 4.1 训练-推理不一致问题

虽然训练时没有使用exp0的prompt_template，但在推理时使用这个模板反而获得了更好的效果。这说明：

1. **Prompt模板的重要性**：明确的输出格式指导可以显著提升模型表现
2. **模型泛化能力**：微调后的模型能够适应不同的提示词格式

### 4.2 格式有效率的提升

- 原始评测：42.69%（大量空输出、截断输出）
- exp0评测：100%（所有输出都有有效内容）

这证明了**推理配置对输出质量的影响**远大于训练本身。

---

## 5. 结论

1. **微调本身是有效的**：使用正确的推理配置后，模型表现显著提升
2. **问题在于推理配置**：原始评测使用的配置导致模型无法正常生成
3. **建议**：后续评测统一使用exp0的评测模式和提示词格式
