# GeoKD-SR 四级审查脚本使用指南

## 概述

`validate_dataset_v2.py` 是GeoKD-SR数据集的四级审查脚本，实现了30项检查的完整审查体系。

## 快速开始

### 基本用法

```bash
# 审查所有层级
python scripts/validate_dataset_v2.py \
    --input data/geosr_chain/balanced_topology.jsonl

# 只审查L1和L2
python scripts/validate_dataset_v2.py \
    --input data/geosr_chain/balanced_topology.jsonl \
    --levels 1,2

# 输出JSON格式
python scripts/validate_dataset_v2.py \
    --input data/geosr_chain/balanced_topology.jsonl \
    --format json
```

## 命令行参数

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| --input | -i | 输入JSONL文件路径 | 必需 |
| --output | -o | 输出目录 | outputs |
| --levels | -l | 审查层级 (1,2,3,4) | 1,2,3,4 |
| --format | -f | 输出格式 (markdown/json/csv) | markdown |
| --config | -c | 配置文件路径 | 无 |

## 审查层级详解

### L1: 格式审查 (4项)

| ID | 检查项 | 说明 |
|----|--------|------|
| L1-1 | 字段完整性 | 检查11个必需字段 |
| L1-2 | 数据类型 | 验证字段类型正确 |
| L1-3 | ID唯一性 | 确保无重复ID |
| L1-4 | JSON格式 | 验证JSON有效性 |

### L2: 逻辑审查 (8项)

| ID | 检查项 | 说明 |
|----|--------|------|
| L2-5 | 推理链步数 | 必须为5步 |
| L2-6 | 推理链字段 | 每步必需字段检查 |
| L2-7 | 坐标有效性 | 中国境内坐标 (73-135E, 18-54N) |
| L2-8 | 坐标一致性 | 推理链与实体坐标一致 |
| L2-9 | difficulty一致性 | difficulty与score映射正确 |
| L2-10 | 答案逻辑 | 是否类问题有明确判断 |
| L2-11 | 距离准确性 | 距离计算误差<10% |
| L2-12 | entity_to_token | 映射完整且正确 |

### L3: 分布审查 (8项)

| ID | 检查项 | 目标值 | 允许偏差 |
|----|--------|--------|----------|
| L3-13 | directional分布 | 25% | ±5% |
| L3-14 | topological分布 | 27.5% | ±5% |
| L3-15 | metric分布 | 27.5% | ±5% |
| L3-16 | composite分布 | 20% | ±5% |
| L3-17 | easy难度分布 | 30% | ±5% |
| L3-18 | medium难度分布 | 50% | ±5% |
| L3-19 | hard难度分布 | 20% | ±5% |
| L3-20 | 实体分布CV | <0.7 | - |

### L4: 语义审查 (10项)

| ID | 检查项 | 说明 |
|----|--------|------|
| L4-21 | Within关键词 | "位于/内/境内" |
| L4-22 | Contains关键词 | "包含/含有" |
| L4-23 | Adjacent关键词 | "相邻/接壤/毗邻" |
| L4-24 | Disjoint关键词 | "不相邻/分离/相离" |
| L4-25 | Overlap关键词 | "流经/贯穿/跨越" |
| L4-26 | 方向表达统一 | 8方向格式 |
| L4-27 | spatial_tokens覆盖 | 出现在问题中 |
| L4-28 | 提示词分布 | 无过度集中 |
| L4-29 | 省份覆盖 | 至少30/34省 |
| L4-30 | 问题多样性 | 模板不重复率 |

## 输出格式

### Markdown格式
默认格式，包含详细的审查结果和问题列表。

### JSON格式
结构化数据，便于程序处理：
```json
{
  "validation_time": "2026-03-08T19:16:21",
  "data_file": "data/geosr_chain/balanced_topology.jsonl",
  "total_records": 6522,
  "overall_pass_rate": 68.8,
  "level_reports": {...},
  "statistics": {...},
  "issues": [...]
}
```

### CSV格式
问题列表，便于Excel分析。

## 返回码

| 返回码 | 含义 |
|--------|------|
| 0 | 审查通过，无严重问题 |
| 1 | 存在重要问题，建议修复 |
| 2 | 存在严重问题，必须修复 |

## 常见问题

### Q: 如何只检查分布？
```bash
python scripts/validate_dataset_v2.py \
    --input data.jsonl \
    --levels 3
```

### Q: 如何导出问题列表用于修复？
```bash
python scripts/validate_dataset_v2.py \
    --input data.jsonl \
    --format csv
```

### Q: 如何自定义审查阈值？
编辑 `config/validation_config.yaml` 文件。
