### 1.2 关系分类

**基础关系（2 类）：**
- **方向推理 (directional)**：基于两实体质心计算八方位方向及方位角
- **距离推理 (metric)**：基于两实体质心计算直线距离

**拓扑关系（5 类，各有正/负例）：**
- **contains** / **within** / **touches** / **crosses** / **disjoint**

**复合关系（4 类）：**
- **direction_distance**：方向 + 距离
- **direction_topology**：方向 + 拓扑
- **distance_topology**：距离 + 拓扑
- **direction_distance_topology**：三重复合

## 2. 调用流程

完整的 8 步生成流程：

```
实体对数据 (JSONL)
       |
       v
[1] 读取 JSONL 文件 (pairs_positive.jsonl / pairs_negative.jsonl)
       |
       v
[2] 根据 target_relation 和 is_negative 选择对应模板
       |
       v
[3] 准备模板变量 (从 JSONL 数据行映射到模板占位符)
       |
       v
[4] 渲染模板 (替换 {{variable}} 占位符为实际值)
       |
       v
[5] 调用 Qwen API (通过 DashScope 兼容模式)
       |
       v
[6] 解析返回的 JSON 结果
       |
       v
[7] 验证 answer_structured 与 spatial_facts 的一致性
       |
       v
[8] 写入输出文件 (instances.jsonl)
```

## 3. 变量映射

### 3.1 实际数据格式

**正例 (pairs_positive.jsonl)：**

```json
{
  "pair_id": "dir_00001",
  "target_relation": "directional",
  "entity_a": {"name_zh": "武汉市", "centroid": [114.31, 30.59], "type": "city"},
  "entity_b": {"name_zh": "长沙市", "centroid": [112.94, 28.23], "type": "city"},
  "spatial_facts": {"direction_8": "东北", "azimuth_deg": 29.7},
  "reference_entity": "entity_b"
}
```

**负例 (pairs_negative.jsonl)：**

```json
{
  "pair_id": "neg_contains_00001",
  "target_relation": "topological.contains",
  "is_negative": true,
  "entity_a": {"name_zh": "河南省", "centroid": [113.65, 34.76], "type": "province"},
  "entity_b": {"name_zh": "武汉市", "centroid": [114.31, 30.59], "type": "city"},
  "spatial_facts": {"contains": false},
  "reference_entity": "entity_a"
}
```

**关键注意事项：**
- 数据中 entity 只有 `name_zh` / `centroid` / `type`，**没有 `geometry_type`**
- 数据中**没有 `actual_container`** 字段（负例模板中引用的 `actual_container` 需由模型推理生成或另行提供）
- `centroid` 是数组 `[lng, lat]`，转字符串时保留数组格式
- 负例有 `is_negative` 字段
- 有 `reference_entity` 字段
- `spatial_facts` 的键名因关系类型不同而不同

### 3.2 变量列表

| 变量 | 来源 | 示例值 |
|------|------|--------|
| `pair_id` | `pair["pair_id"]` | `"dir_00142"` |
| `entity_a_name` | `entity_a["name_zh"]` | `"武汉市"` |
| `entity_a_type` | `entity_a["type"]` | `"city"` |
| `entity_a_centroid` | `str(entity_a["centroid"])` | `"[114.31, 30.59]"` |
| `entity_b_name` | `entity_b["name_zh"]` | `"长沙市"` |
| `entity_b_type` | `entity_b["type"]` | `"city"` |
| `entity_b_centroid` | `str(entity_b["centroid"])` | `"[112.94, 28.23]"` |
| `fact_direction_8` | `spatial_facts.get("direction_8", "")` | `"东北"` |
| `fact_azimuth_deg` | `str(spatial_facts.get("azimuth_deg", ""))` | `"29.7"` |
| `fact_distance_km` | `str(spatial_facts.get("distance_km", ""))` | `"296"` |
| `fact_relation` | 对应拓扑字段值（见下方提取逻辑） | `"true"` / `"false"` |
| `difficulty` | 调用方计算 | `"easy"` |

> **关于 `geometry_type` 和 `actual_container`**：模板文件中引用了 `{{entity_a_geometry_type}}`、`{{entity_b_geometry_type}}` 和 `{{actual_container}}` 占位符，但实际 JSONL 数据中不包含这些字段。`prepare_template_data()` 函数将使用空字符串作为默认值，模型可根据实体类型自行推断几何类型。

### 3.3 prepare_template_data() 函数

```python
def prepare_template_data(pair: dict, difficulty: str) -> dict:
    """从 JSONL 数据行准备模板替换所需的变量字典。

    注意：
    - 数据中 entity 没有 geometry_type，设为空字符串
    - 数据中没有 actual_container，设为空字符串
    - centroid 是数组 [lng, lat]，转字符串保留数组格式
    - spatial_facts 的键名因关系类型不同而不同
    """
    entity_a = pair["entity_a"]
    entity_b = pair["entity_b"]
    spatial_facts = pair["spatial_facts"]

    # 提取拓扑关系值：遍历所有可能的拓扑键，取第一个非空值
    topo_keys = ["contains", "within", "touches", "crosses", "disjoint"]
    fact_relation = ""
    for key in topo_keys:
        if key in spatial_facts:
            fact_relation = str(spatial_facts[key]).lower()
            break

    return {
        # 实体对标识
        "pair_id": pair["pair_id"],

        # 实体 A 信息（数据中无 geometry_type）
        "entity_a_name": entity_a["name_zh"],
        "entity_a_type": entity_a["type"],
        "entity_a_centroid": str(entity_a["centroid"]),
        "entity_a_geometry_type": "",  # 数据中不存在，留空

        # 实体 B 信息（数据中无 geometry_type）
        "entity_b_name": entity_b["name_zh"],
        "entity_b_type": entity_b["type"],
        "entity_b_centroid": str(entity_b["centroid"]),
        "entity_b_geometry_type": "",  # 数据中不存在，留空

        # 空间事实 - 方向
        "fact_direction_8": spatial_facts.get("direction_8", ""),
        "fact_azimuth_deg": str(spatial_facts.get("azimuth_deg", "")),

        # 空间事实 - 距离
        "fact_distance_km": str(spatial_facts.get("distance_km", "")),

        # 空间事实 - 拓扑关系
        "fact_relation": fact_relation,

        # 难度（由调用方通过 DifficultyBalancer 计算）
        "difficulty": difficulty,

        # 负例辅助字段（数据中不存在 actual_container，留空）
        "actual_container": "",
    }
```

### 3.4 模板渲染函数

```python
import re

def render_template(template_str: str, data: dict) -> str:
    """替换模板中的所有 {{variable}} 占位符。

    未在 data 中找到的变量将保留原始占位符不变。
    """
    def replacer(match):
        key = match.group(1)
        value = data.get(key, match.group(0))
        if isinstance(value, (list, dict)):
            return str(value)
        return str(value)
    return re.sub(r'\{\{(\w+)\}\}', replacer, template_str)
```

### 4.2 get_template_path() 函数

```python
import os

# target_relation 到模板文件名的映射
TEMPLATE_MAP = {
    "directional": {
        "positive": "directional_positive.md",
    },
    "metric": {
        "positive": "metric_positive.md",
    },
    "topological.contains": {
        "positive": "topo_contains_positive.md",
        "negative": "topo_contains_negative.md",
    },
    "topological.within": {
        "positive": "topo_within_positive.md",
        "negative": "topo_within_negative.md",
    },
    "topological.touches": {
        "positive": "topo_touches_positive.md",
        "negative": "topo_touches_negative.md",
    },
    "topological.crosses": {
        "positive": "topo_crosses_positive.md",
        "negative": "topo_crosses_negative.md",
    },
    "topological.disjoint": {
        "positive": "topo_disjoint_positive.md",
        "negative": "topo_disjoint_negative.md",
    },
    "composite.direction_distance": {
        "positive": "composite_dd_positive.md",
    },
    "composite.direction_topology": {
        "positive": "composite_dir_topo_positive.md",
        "negative": "composite_dir_topo_negative.md",
    },
    "composite.distance_topology": {
        "positive": "composite_dist_topo_positive.md",
        "negative": "composite_dist_topo_negative.md",
    },
    "composite.direction_distance_topology": {
        "positive": "composite_ddt_positive.md",
        "negative": "composite_ddt_negative.md",
    },
}


def get_template_path(target_relation: str, is_negative: bool, prompts_dir: str = "prompts") -> str:
    """根据 target_relation 和 is_negative 返回模板文件路径。

    Args:
        target_relation: 关系类型，如 "directional"、"topological.contains"
        is_negative: 是否为负例
        prompts_dir: 模板文件所在目录

    Returns:
        模板文件的绝对/相对路径

    Raises:
        KeyError: target_relation 不在映射表中
        ValueError: 该关系类型不支持负例模板但 is_negative=True
    """
    if target_relation not in TEMPLATE_MAP:
        raise KeyError(f"未知的 target_relation: {target_relation}，"
                       f"可选值: {list(TEMPLATE_MAP.keys())}")

    polarity = "negative" if is_negative else "positive"
    templates = TEMPLATE_MAP[target_relation]

    if polarity not in templates:
        if is_negative:
            raise ValueError(f"关系类型 '{target_relation}' 不支持负例模板")
        else:
            raise ValueError(f"关系类型 '{target_relation}' 缺少正例模板")

    filename = templates[polarity]
    return os.path.join(prompts_dir, filename)


def load_template(target_relation: str, is_negative: bool, prompts_dir: str = "prompts") -> str:
    """加载并返回模板内容。"""
    path = get_template_path(target_relation, is_negative, prompts_dir)
    with open(path, "r", encoding="utf-8") as f:
        return f.read()
```

## 5. 难度比例控制

### 5.1 目标比例

| 难度级别 | 目标占比 |
|---------|---------|
| easy | 30% |
| medium | 50% |
| hard | 20% |

### 5.2 DifficultyBalancer 类

```python
from collections import Counter


class DifficultyBalancer:
    """难度均衡分配器，确保生成题目中各难度级别比例接近目标比例。

    使用偏差优先策略：每次分配时选择当前实际比例与目标比例偏差最大的级别。
    """

    def __init__(self, target_ratios: dict = None):
        """
        Args:
            target_ratios: 难度级别到目标比例的映射，默认 {"easy": 0.30, "medium": 0.50, "hard": 0.20}
        """
        self.counter = Counter()
        self.target_ratios = target_ratios or {"easy": 0.30, "medium": 0.50, "hard": 0.20}

    def get_next(self) -> str:
        """获取下一个应分配的难度级别。

        Returns:
            难度级别字符串: "easy" / "medium" / "hard"
        """
        total = sum(self.counter.values())
        if total == 0:
            # 第一条，选择目标比例最高的级别
            return max(self.target_ratios, key=self.target_ratios.get)

        # 计算每个级别的偏差（目标比例 - 实际比例），选择偏差最大的
        deviations = {}
        for level, target in self.target_ratios.items():
            actual = self.counter[level] / total
            deviations[level] = target - actual

        return max(deviations, key=deviations.get)

    def record(self, difficulty: str):
        """记录已分配的难度级别。"""
        self.counter[difficulty] += 1

    def get_stats(self) -> dict:
        """获取当前分配统计。"""
        total = sum(self.counter.values())
        if total == 0:
            return {"total": 0, "distribution": {}}
        return {
            "total": total,
            "distribution": {
                level: {"count": count, "ratio": f"{count / total:.1%}"}
                for level, count in sorted(self.counter.items())
            },
        }
```

### 5.3 使用示例

```python
balancer = DifficultyBalancer()

for pair in entity_pairs:
    difficulty = balancer.get_next()

    # ... 生成题目 ...

    balancer.record(difficulty)

# 查看统计
print(balancer.get_stats())
# 输出示例: {'total': 100, 'distribution': {'easy': {'count': 30, 'ratio': '30.0%'}, ...}}
```

## 6. Qwen API 调用

### 6.1 接入方式

使用阿里 Qwen API，通过 **DashScope 兼容模式** 调用 OpenAI SDK。

```python
from openai import OpenAI

# 初始化客户端（DashScope 兼容模式）
client = OpenAI(
    api_key="your-dashscope-api-key",
    base_url="https://dashscope.aliyuncs.com/compatible-mode/v1",
)
```

### 6.2 模型选择

| 模型 | 说明 | 适用场景 |
|------|------|---------|
| `qwen-plus` | 推荐默认模型，性价比高 | 批量生成 |
| `qwen-max` | 效果最优，精度高 | 高质量要求 |
| `qwen-turbo` | 速度最快，成本低 | 快速测试 |

### 6.3 调用函数

```python
SYSTEM_PROMPT = "你是一个地理空间推理数据集生成专家。请严格按照要求输出JSON格式。"


def call_qwen_api(prompt_text: str, model: str = "qwen-plus") -> str:
    """调用 Qwen API 并返回原始文本结果。

    Args:
        prompt_text: 渲染后的完整 prompt 文本
        model: 模型名称，默认 qwen-plus

    Returns:
        API 返回的文本内容（JSON 字符串）
    """
    response = client.chat.completions.create(
        model=model,
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {"role": "user", "content": prompt_text},
        ],
        temperature=0.7,
        top_p=0.9,
        max_tokens=4096,
        extra_body={"response_format": {"type": "json_object"}},
    )

    return response.choices[0].message.content
```

### 6.4 参数说明

| 参数 | 推荐值 | 说明 |
|------|--------|------|
| `temperature` | 0.7 | 保证多样性的同时维持稳定性 |
| `top_p` | 0.9 | nucleus sampling |
| `max_tokens` | 4096 | 足够容纳4种题型的JSON |
| `extra_body.response_format` | `{"type": "json_object"}` | 强制 JSON 输出 |

## 7. 失败处理

### 7.1 重试策略

| 失败类型 | 最大重试次数 | 重试操作 |
|---------|------------|---------|
| JSON 解析失败 | 1 次 | 追加格式提醒后重试 |
| Schema 验证失败 | 1 次 | 追加字段提醒后重试 |
| 两次均失败 | -- | 记录 error.log，跳过该条 |

### 7.2 call_with_retry() 函数

```python
import json
import logging
from datetime import datetime

logger = logging.getLogger(__name__)

# 必须包含的顶级字段
REQUIRED_QUESTION_TYPES = ["true_false", "choice", "fill_blank", "open_qa"]


def call_with_retry(prompt: str, pair_id: str, model: str = "qwen-plus") -> dict | None:
    """带重试机制的 LLM 调用。

    JSON 解析失败：重试 1 次，追加格式提醒。
    Schema 验证失败：重试 1 次，追加字段提醒。
    两次失败：记录 error.log，返回 None。

    Args:
        prompt: 渲染后的 prompt 文本
        pair_id: 实体对标识，用于日志
        model: Qwen 模型名称

    Returns:
        解析后的 dict，失败返回 None
    """
    current_prompt = prompt

    for attempt in range(2):
        try:
            # 调用 API
            raw = call_qwen_api(current_prompt, model=model)

            # 尝试解析 JSON
            result = json.loads(raw)

            # Schema 验证：检查必要字段
            for qtype in REQUIRED_QUESTION_TYPES:
                if qtype not in result:
                    raise ValueError(f"缺少必要字段: {qtype}")
                q = result[qtype]
                if "question" not in q:
                    raise ValueError(f"'{qtype}' 中缺少 'question' 字段")
                if "answer_structured" not in q:
                    raise ValueError(f"'{qtype}' 中缺少 'answer_structured' 字段")

            # 选择题额外验证
            if "options" not in result.get("choice", {}):
                raise ValueError("选择题缺少 'options' 字段")

            return result

        except json.JSONDecodeError as e:
            logger.warning(f"[{pair_id}] JSON 解析失败 (尝试 {attempt + 1}/2): {e}")
            if attempt == 0:
                # 追加格式提醒
                current_prompt = prompt + "\n\n【重要提醒】请确保输出为纯JSON格式，不要添加markdown代码块标记（如 ```json），直接输出JSON文本。"
            else:
                log_error(pair_id, current_prompt, raw)
                return None

        except ValueError as e:
            logger.warning(f"[{pair_id}] Schema 验证失败 (尝试 {attempt + 1}/2): {e}")
            if attempt == 0:
                # 追加字段提醒
                current_prompt = prompt + (
                    "\n\n【重要提醒】输出必须包含 true_false、choice、fill_blank、open_qa 四个字段。"
                    "每个字段必须包含 question 和 answer_structured 子字段。"
                    "选择题必须包含 options 字段且至少2个选项。"
                )
            else:
                log_error(pair_id, current_prompt, raw if 'raw' in dir() else "")
                return None

    return None


def log_error(pair_id: str, prompt: str, raw_output: str):
    """记录失败的调用详情到 error.log 文件。"""
    error_record = {
        "pair_id": pair_id,
        "timestamp": datetime.now().isoformat(),
        "prompt_length": len(prompt),
        "raw_output": raw_output[:500],  # 截取前500字符
    }
    with open("error.log", "a", encoding="utf-8") as f:
        f.write(json.dumps(error_record, ensure_ascii=False) + "\n")

## 8. 输出验证

### 8.1 answer_structured 与 spatial_facts 一致性检查

```python
def validate_instance(instance: dict, pair: dict) -> tuple[bool, list[str]]:
    """验证生成结果中 answer_structured 与 spatial_facts 的一致性。

    Args:
        instance: LLM 生成的结果 dict
        pair: 原始实体对数据

    Returns:
        (is_valid, error_messages) 元组
    """
    facts = pair["spatial_facts"]
    errors = []

    # 遍历所有题型检查一致性
    for qtype in REQUIRED_QUESTION_TYPES:
        if qtype not in instance:
            continue

        q = instance[qtype]
        ans = q.get("answer_structured", {})
        if not ans:
            continue

        prefix = f"[{qtype}]"

        # 方向一致性
        if "direction" in ans and "direction_8" in facts:
            if ans["direction"] != facts["direction_8"]:
                errors.append(f"{prefix} direction 不一致: "
                              f"answer='{ans['direction']}', fact='{facts['direction_8']}'")

        # 距离一致性（允许1公里误差）
        if "distance_km" in ans and "distance_km" in facts:
            try:
                if abs(float(ans["distance_km"]) - float(facts["distance_km"])) > 1:
                    errors.append(f"{prefix} distance_km 不一致: "
                                  f"answer={ans['distance_km']}, fact={facts['distance_km']}")
            except (TypeError, ValueError):
                pass

        # 拓扑一致性
        for topo_key in ["contains", "within", "touches", "crosses", "disjoint"]:
            if topo_key in ans and topo_key in facts:
                if bool(ans[topo_key]) != bool(facts[topo_key]):
                    errors.append(f"{prefix} {topo_key} 不一致: "
                                  f"answer={ans[topo_key]}, fact={facts[topo_key]}")

    if errors:
        logger.warning(f"验证失败 [{pair['pair_id']}]: {'; '.join(errors)}")

    return (len(errors) == 0, errors)
```

### 8.2 JSON 结构验证

```python
def validate_json_structure(result: dict) -> tuple[bool, list[str]]:
    """验证返回 JSON 结构的完整性。

    Returns:
        (is_valid, error_messages) 元组
    """
    errors = []

    for key in REQUIRED_QUESTION_TYPES:
        if key not in result:
            errors.append(f"缺少顶级字段 '{key}'")
            continue

        q = result[key]

        # 每个题型应包含的基本字段
        if "question" not in q:
            errors.append(f"'{key}' 中缺少 'question'")
        if "answer_structured" not in q:
            errors.append(f"'{key}' 中缺少 'answer_structured'")

    # 选择题额外验证
    choice = result.get("choice", {})
    if "options" not in choice:
        errors.append("选择题缺少 'options' 字段")
    elif len(choice["options"]) < 2:
        errors.append("选择题选项数量不足（至少2个）")

    return (len(errors) == 0, errors)
```

### 8.3 完整验证流程

```python
def full_validate(instance: dict, pair: dict) -> tuple[bool, list[str]]:
    """完整验证流程：结构验证 + 一致性验证。

    Returns:
        (is_valid, all_errors) 元组
    """
    all_errors = []

    # 1. JSON 结构验证
    struct_ok, struct_errors = validate_json_structure(instance)
    all_errors.extend(struct_errors)

    # 2. 空间事实一致性验证
    if struct_ok:  # 结构完整才检查一致性
        consistent, consist_errors = validate_instance(instance, pair)
        all_errors.extend(consist_errors)

    return (len(all_errors) == 0, all_errors)

def load_pairs(filepath: str) -> list[dict]:
    """从 JSONL 文件加载实体对数据。"""
    pairs = []
    with open(filepath, "r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if line:
                pairs.append(json.loads(line))
    return pairs


def render_template(template_str: str, data: dict) -> str:
    """替换模板中的 {{variable}} 占位符。"""
    def replacer(match):
        key = match.group(1)
        value = data.get(key, match.group(0))
        return str(value) if not isinstance(value, (list, dict)) else str(value)
    return re.sub(r"\{\{(\w+)\}\}", replacer, template_str)


def load_template(target_relation: str, is_negative: bool) -> str:
    """加载模板内容。"""
    polarity = "negative" if is_negative else "positive"
    templates = TEMPLATE_MAP[target_relation]
    if polarity not in templates:
        raise ValueError(f"关系类型 '{target_relation}' 不支持{polarity}模板")
    path = os.path.join(PROMPTS_DIR, templates[polarity])
    with open(path, "r", encoding="utf-8") as f:
        return f.read()


def prepare_template_data(pair: dict, difficulty: str) -> dict:
    """从 JSONL 数据行映射到模板变量。"""
    entity_a = pair["entity_a"]
    entity_b = pair["entity_b"]
    spatial_facts = pair["spatial_facts"]

    # 提取拓扑关系值
    topo_keys = ["contains", "within", "touches", "crosses", "disjoint"]
    fact_relation = ""
    for key in topo_keys:
        if key in spatial_facts:
            fact_relation = str(spatial_facts[key]).lower()
            break

    return {
        "pair_id": pair["pair_id"],
        "entity_a_name": entity_a["name_zh"],
        "entity_a_type": entity_a["type"],
        "entity_a_centroid": str(entity_a["centroid"]),

def call_qwen_api(prompt_text: str) -> str:
    """调用 Qwen API。"""
    response = client.chat.completions.create(
        model=MODEL_NAME,

            {"role": "user", "content": prompt_text},
        ],
        temperature=0.7,
        top_p=0.9,
        max_tokens=4096,
        extra_body={"response_format": {"type": "json_object"}},
    )
    return response.choices[0].message.content

def full_validate(instance: dict, pair: dict) -> tuple[bool, list[str]]:

    all_errors.extend(struct_errors)
    if struct_ok:
        _, consist_errors = validate_instance(instance, pair)
        all_errors.extend(consist_errors)
    return (len(all_errors) == 0, all_errors)


def call_with_retry(prompt: str, pair_id: str) -> dict | None:
    """带重试的 API 调用（最多2次）。"""
    current_prompt = prompt

    for attempt in range(2):
        try:
            raw = call_qwen_api(current_prompt)
            result = json.loads(raw)

            # Schema 验证
            for qtype in REQUIRED_QUESTION_TYPES:
                if qtype not in result:
                    raise ValueError(f"缺少: {qtype}")

            return result

        except json.JSONDecodeError as e:
            logger.warning(f"[{pair_id}] JSON解析失败 (尝试 {attempt + 1}/2): {e}")
            if attempt == 0:
                current_prompt = prompt + (
                    "\n\n【重要提醒】请确保输出为纯JSON格式，"
                    "不要添加markdown代码块标记，直接输出JSON文本。"
                )
            else:
                log_error(pair_id, prompt, raw)
                return None

        except ValueError as e:
            logger.warning(f"[{pair_id}] Schema验证失败 (尝试 {attempt + 1}/2): {e}")
            if attempt == 0:
                current_prompt = prompt + (
                    "\n\n【重要提醒】输出必须包含 true_false、choice、fill_blank、open_qa "
                    "四个字段，每个字段包含 question 和 answer_structured。"
                )
            else:
                log_error(pair_id, prompt, raw if 'raw' in dir() else "")
                return None

    return None


def log_error(pair_id: str, prompt: str, raw_output: str):

    error_record = {
        "pair_id": pair_id,
        "timestamp": datetime.now().isoformat(),
        "prompt_length": len(prompt),
        "raw_output": (raw_output or "")[:500],
    }
    with open(ERROR_LOG_FILE, "a", encoding="utf-8") as f:
        f.write(json.dumps(error_record, ensure_ascii=False) + "\n")

# 难度均衡器

class DifficultyBalancer:
    """难度均衡分配器: easy 30% / medium 50% / hard 20%"""

    def __init__(self, target_ratios: dict = None):
        self.counter = Counter()
        self.target_ratios = target_ratios or DIFFICULTY_RATIOS

    def get_next(self) -> str:
        total = sum(self.counter.values())
        if total == 0:
            return max(self.target_ratios, key=self.target_ratios.get)
        deviations = {
            level: target - self.counter[level] / total
            for level, target in self.target_ratios.items()
        }
        return max(deviations, key=deviations.get)

    def record(self, difficulty: str):
        self.counter[difficulty] += 1

    def get_stats(self) -> dict:
        total = sum(self.counter.values())
        if total == 0:
            return {"total": 0, "distribution": {}}
        return {
            "total": total,
            "distribution": {
                level: {"count": count, "ratio": f"{count / total:.1%}"}
                for level, count in sorted(self.counter.items())
            },
        }

def generate_summary(output_file: str):
    """生成输出文件的统计摘要。"""
    relation_counter = Counter()
    difficulty_counter = Counter()
    polarity_counter = Counter()
    total = 0

    try:
        with open(output_file, "r", encoding="utf-8") as f:
            for line in f:
                line = line.strip()
                if not line:
                    continue
                instance = json.loads(line)
                total += 1
                relation_counter[instance.get("target_relation", "unknown")] += 1
                difficulty_counter[instance.get("difficulty", "unknown")] += 1
                polarity_counter["负例" if instance.get("is_negative") else "正例"] += 1
    except FileNotFoundError:
        logger.warning(f"输出文件不存在: {output_file}")
        return

    print("\n" + "=" * 50)
    print("           生成结果统计")
    print("=" * 50)
    print(f"总计: {total} 条\n")

    print("--- 关系类型分布 ---")
    for relation, count in relation_counter.most_common():
        print(f"  {relation}: {count} ({count / total:.1%})")

    print("\n--- 难度分布 ---")
    for difficulty in ["easy", "medium", "hard"]:
        count = difficulty_counter.get(difficulty, 0)
        print(f"  {difficulty}: {count} ({count / total:.1%})")

    print("\n--- 正负例分布 ---")
    for polarity, count in polarity_counter.items():
        print(f"  {polarity}: {count} ({count / total:.1%})")

    print("=" * 50)

def batch_generate():
    """完整批量生成 Pipeline。"""

    # 确保输出目录存在
    os.makedirs(os.path.dirname(OUTPUT_FILE), exist_ok=True)

    # 加载断点续传进度
    processed_ids = load_progress()
    logger.info(f"已加载断点续传进度: {len(processed_ids)} 条已处理")

    # 加载所有输入数据
    all_pairs = []
    for filepath in INPUT_FILES:
        if os.path.exists(filepath):
            pairs = load_pairs(filepath)
            logger.info(f"加载 {filepath}: {len(pairs)} 条")
            all_pairs.extend(pairs)
        else:
            logger.warning(f"文件不存在，跳过: {filepath}")

    # 过滤已处理的
    pending_pairs = [p for p in all_pairs if p["pair_id"] not in processed_ids]
    total = len(pending_pairs)
    logger.info(f"待处理: {total} 条（已跳过 {len(processed_ids)} 条）")

    if total == 0:
        logger.info("没有待处理的数据，退出。")
        return

    # 初始化
    balancer = DifficultyBalancer()
    min_interval = 60.0 / RATE_LIMIT_PER_MINUTE  # 速率控制间隔
    success_count = 0
    fail_count = 0
    skip_count = 0

    for i, pair in enumerate(pending_pairs):
        pair_id = pair["pair_id"]
        target_relation = pair["target_relation"]
        is_negative = pair.get("is_negative", False)

        # 检查模板是否存在
        try:
            template = load_template(target_relation, is_negative)
        except (KeyError, ValueError) as e:
            logger.warning(f"[{pair_id}] 跳过 - 模板不可用: {e}")
            skip_count += 1
            continue

        # 获取难度并准备数据
        difficulty = balancer.get_next()
        template_data = prepare_template_data(pair, difficulty)

        # 渲染模板
        prompt = render_template(template, template_data)

        # 调用 LLM（带重试）
        try:
            result = call_with_retry(prompt, pair_id)

            if result is None:
                fail_count += 1
                continue

            # 验证结果
            is_valid, errors = full_validate(result, pair)
            if not is_valid:
                logger.warning(f"[{pair_id}] 验证失败: {'; '.join(errors)}")
                fail_count += 1
                continue

            # 构建完整实例记录
            instance = {
                "pair_id": pair_id,
                "target_relation": target_relation,
                "is_negative": is_negative,
                "difficulty": difficulty,
                "entity_a": pair["entity_a"]["name_zh"],
                "entity_b": pair["entity_b"]["name_zh"],
                "spatial_facts": pair["spatial_facts"],
                **result,
            }

            # 写入输出文件
            with open(OUTPUT_FILE, "a", encoding="utf-8") as f:
                f.write(json.dumps(instance, ensure_ascii=False) + "\n")

            # 更新状态
            balancer.record(difficulty)
            processed_ids.add(pair_id)
            success_count += 1

        except Exception as e:
            logger.error(f"[{pair_id}] 生成失败: {e}")
            fail_count += 1

        # 进度追踪：每50条输出一次
        processed = success_count + fail_count + skip_count
        if processed % PROGRESS_INTERVAL == 0:
            save_progress(processed_ids)
            logger.info(
                f"进度: {processed}/{total} | "
                f"成功: {success_count} | 失败: {fail_count} | 跳过: {skip_count} | "
                f"难度: {balancer.get_stats()['distribution']}"
            )

        # 速率控制
        time.sleep(min_interval)

    # 最终保存进度
    save_progress(processed_ids)

    # 输出汇总
    logger.info("=" * 60)
    logger.info("生成完成!")
    logger.info(f"  总计: {total} | 成功: {success_count} | 失败: {fail_count} | 跳过: {skip_count}")
    logger.info(f"  难度分布: {balancer.get_stats()['distribution']}")
    logger.info("=" * 60)

    # 打印详细统计
    generate_summary(OUTPUT_FILE)


# ============================================================
# 入口
# ============================================================

if __name__ == "__main__":
    batch_generate()
