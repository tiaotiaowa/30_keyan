**🔍 GeoSignal数据集详细构建方法（深入分析）**：

| 信号类型 | 描述 | 数据来源 | 对应任务 |
|---------|------|----------|----------|
| **G1: Paper Content** | 论文标题、摘要、全文 | DDE Scholar, GAKG | 摘要生成 |
| **G2: Category** | 论文/术语分类 | DDE Scholar, Wikipedia | 文本分类 |
| **G5: Entity Mentions** | 实体提及 | GAKG, Wikipedia | 命名实体识别 |
| **G6: Relations** | 实体间关系 | GAKG, GSO | **关系推理** ⭐ |
| **G10: Factual Knowledge** | 事实知识 | QA平台 | 事实问答与验证 |

**数据构建三种方法**：
1. **重新结构化**：从现有数据库提取结构化信号，转换为 `<input, output>` 对
2. **自指令生成**：使用GPT-4从18个子领域生成18,000个QA对，专家验证
3. **半自动化流程**：爬取数据库 + OpenAI生成模板 + 人类专家清洗

**训练超参数（可直接借鉴）**：
| 参数 | 预训练 | 指令微调 | 工具学习 |
|------|--------|---------|---------|
| 学习率 | 1e-5 | 1e-4 | 1e-5 |
| 批次大小 | 128 | 128 | - |
| LoRA rank | - | 8 | 8 |
| 可训练参数 | - | 6M | 4M |
| GPU | 4×A100-40GB | 1×RTX 3090 | 8×RTX 3090 |

**关键发现：最佳训练顺序**
```
Step 1: 进一步预训练 → GeoLLaMA
```

# 第一部分：研究概述

## 1.1 研究背景

### 地理人工智能的发展趋势

将大型语言模型（LLMs）集成到地理信息系统（GIS）中，标志着地理人工智能（GeoAI）领域的一次革命性转变。随着空间数据体量和复杂性的持续增长，LLMs为空间分析自动化、自然语言交互实现以及GIS环境中的分析推理增强提供了新范式。

然而，当前LLM4GIS研究大多依赖于云端大规模模型，其在**端侧设备**上的部署面临严峻的算力与能耗约束。现有端侧方案多采用单一压缩模型，在需要深度**地图认知、空间关系推理与地学知识融合**的任务上性能受限。

### 2025年关键发现

**重要发现1：大模型的"虚假空间智能"问题**
- 2025年OSI-Bench揭示：大模型在空间推理基准上的高分可能是"背答案"而非真正习得空间推理能力
- 训练数据与测试数据同源问题严重

**重要发现2：知识蒸馏在推理任务上的有效性**
- DeepSeek-R1（2025）验证：蒸馏在推理能力迁移上优于强化学习
- 斯坦福团队：思维链蒸馏(CoT-Distill)可有效迁移推理能力

### 研究空白

| 空白领域 | 现状 | 机会 |
|---------|------|------|
| GIS领域知识蒸馏 | 仅2篇论文，均聚焦遥感图像 | **空间推理能力蒸馏**尚未被研究 |
| 空间认知蒸馏 | 无系统研究 | 首次探索方向/拓扑/距离推理的蒸馏机制 |
| 地理幻觉抑制 | 无知识图谱引导方法 | 知识图谱校验的蒸馏新范式 |

## 1.2 研究问题与目标

### 核心研究问题

**如何通过知识蒸馏将大模型的地理空间关系推理能力高效迁移到轻量化模型？**

### 子问题

1. 如何设计针对空间关系类型的蒸馏损失函数？
2. 不同类型的空间关系（方向/拓扑/度量）在蒸馏过程中有何差异？
3. 如何评估轻量化模型的地理空间推理能力？

### 分层目标体系

| 层次 | 目标类型 | 具体内容 | 验证标准 |
|------|---------|---------|---------|
| L1 | 空间知识保真 | 地理实体识别、空间关系提取 | 教师模型90%+性能 |
| L2 | 空间推理保真 | 多跳空间推理、方向链推理 | 教师模型85%+性能 |
| L3 | 空间认知保真 | 地理常识推理、地图认知 | 教师模型80%+性能 |

```python
# 逆向KL蒸馏（借鉴DeepSeek-R1）
def reverse_kl_loss(student_logits, teacher_logits, temperature=1.0):
    """
    逆向KL散度：KL(p_student || p_teacher)
    优点：避免模式崩溃，保持生成多样性
    """
    student_probs = F.softmax(student_logits / temperature, dim=-1)
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)
    rkl = torch.sum(student_probs * (torch.log(student_probs) - torch.log(teacher_probs)), dim=-1)
    return rkl.mean()
```

# 第四部分：研究方法

## 4.1 核心创新：空间关系类型感知的蒸馏损失

### 设计思想

不同类型的空间关系在蒸馏过程中应该有不同的权重：
- 方向关系（东、西、南、北）是空间认知的基础，权重最高
- 拓扑关系（包含、相邻）是空间推理的核心，权重较高
- 度量关系（距离、面积）相对容易学习，权重适中

### 核心代码实现

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SpatialRelationAwareDistillationLoss(nn.Module):
    """
    空间关系类型感知的蒸馏损失

    核心思想：不同类型的空间关系在蒸馏过程中应该有不同的权重
    """

    def __init__(self, temperature=2.0):
        super().__init__()
        self.temperature = temperature

        # 空间关系类型权重（可调参）
        self.relation_weights = {
            'directional': 1.5,  # 方向关系权重更高
            'topological': 1.3,  # 拓扑关系
            'metric': 1.0,       # 度量关系
            'order': 1.0,        # 序关系
            'general': 1.0       # 一般文本
        }

    def identify_relation_type(self, text):
        """
        识别文本中的空间关系类型
        使用规则+词表的方法
        """
        # 方向词表
        direction_words = ['东', '西', '南', '北', '东北', '西南', '东南', '西北',
                          '左', '右', '前', '后', '上', '下']
        # 拓扑词表
        topology_words = ['包含', '相邻', '相交', '位于', '穿过', '接壤', '内部']
        # 度量词表
        metric_words = ['距离', '远', '近', '公里', '米', '面积', '长度']

        for word in direction_words:
            if word in text:
                return 'directional'
        for word in topology_words:
            if word in text:
                return 'topological'
        for word in metric_words:
            if word in text:
                return 'metric'

        return 'general'

    def forward(self, teacher_logits, student_logits, input_texts):
        """
        计算空间关系感知的蒸馏损失

        Args:
            teacher_logits: 教师模型的logits [batch_size, seq_len, vocab_size]
            student_logits: 学生模型的logits [batch_size, seq_len, vocab_size]
            input_texts: 输入文本列表 [batch_size]
        """
        total_loss = 0

        for i, text in enumerate(input_texts):
            # 识别空间关系类型
            rel_type = self.identify_relation_type(text)
            weight = self.relation_weights[rel_type]

            # 计算KL散度
            teacher_probs = F.softmax(teacher_logits[i] / self.temperature, dim=-1)
            student_log_probs = F.log_softmax(student_logits[i] / self.temperature, dim=-1)

            kl = F.kl_div(student_log_probs, teacher_probs, reduction='mean')

            # 加权损失
            total_loss += weight * kl

        return total_loss / len(input_texts)
```

## 4.2 辅助创新：动态权重学习（可选增强）

### 设计思想

不使用固定权重，而是根据上下文动态学习权重，增加创新深度，避免被质疑"简单加权"。

### 代码实现

```python
class DynamicWeightLearner(nn.Module):
    """
    根据输入上下文动态学习空间关系权重
    """
    def __init__(self, hidden_dim=768):
        super().__init__()
        self.context_encoder = nn.Sequential(
            nn.Linear(hidden_dim, 256),
            nn.ReLU(),
            nn.Dropout(0.1),
            nn.Linear(256, 64),
            nn.ReLU()
        )

        # 为每种关系类型生成权重
        self.weight_heads = nn.ModuleDict({
            'directional': nn.Linear(64, 1),
            'topological': nn.Linear(64, 1),
            'metric': nn.Linear(64, 1),
            'general': nn.Linear(64, 1)
        })

        # 基础权重（用于残差连接）
        self.base_weights = nn.Parameter(torch.tensor([1.5, 1.3, 1.0, 1.0]))

    def forward(self, hidden_states, relation_types):
        context = self.context_encoder(hidden_states)
        batch_size = hidden_states.size(0)
        weights = torch.zeros(batch_size, device=hidden_states.device)

        for i in range(batch_size):
            rel_idx = relation_types[i].item()
            rel_type = ['directional', 'topological', 'metric', 'general'][rel_idx]
            dynamic_offset = self.weight_heads[rel_type](context[i])
            weights[i] = torch.sigmoid(dynamic_offset) * self.base_weights[rel_idx]

        return weights
```

## 4.3 辅助技术

```python
# 1. 逆向KL蒸馏（借鉴DeepSeek-R1）
def reverse_kl_component(student_logits, teacher_logits, alpha=0.3):
    """
    逆向KL作为辅助损失，避免模式崩溃
    """
    student_probs = F.softmax(student_logits, dim=-1)
    teacher_probs = F.softmax(teacher_logits, dim=-1)
    rkl = torch.sum(student_probs * (torch.log(student_probs + 1e-10) - torch.log(teacher_probs + 1e-10)), dim=-1)
    return alpha * rkl.mean()

# 2. 总损失函数
class TotalDistillationLoss(nn.Module):
    def __init__(self):
        super().__init__()
        self.spatial_loss = SpatialRelationAwareDistillationLoss()
        self.reverse_kl_weight = 0.3
        self.task_loss_weight = 0.5

    def forward(self, teacher_logits, student_logits, input_texts, labels):
        # 空间关系感知损失
        spatial_loss = self.spatial_loss(teacher_logits, student_logits, input_texts)

        # 逆向KL损失
        rkl_loss = reverse_kl_component(student_logits, teacher_logits, self.reverse_kl_weight)

        # 任务特定损失（如问答）
        task_loss = F.cross_entropy(student_logits.view(-1, student_logits.size(-1)), labels.view(-1))

        # 总损失
        total = spatial_loss + rkl_loss + self.task_loss_weight * task_loss
        return total
```

```python
import math
from typing import Tuple

class SpatialDataValidator:
    """空间数据验证器"""

    def validate_direction(self, coords1: Tuple[float, float],
                          coords2: Tuple[float, float],
                          claimed_direction: str) -> dict:
        """验证方向关系是否正确"""
        lat1, lon1 = math.radians(coords1[0]), math.radians(coords1[1])
        lat2, lon2 = math.radians(coords2[0]), math.radians(coords2[1])

        dlon = lon2 - lon1
        x = math.sin(dlon) * math.cos(lat2)
        y = math.cos(lat1) * math.sin(lat2) - math.sin(lat1) * math.cos(lat2) * math.cos(dlon)

        azimuth = (math.degrees(math.atan2(x, y)) + 360) % 360
        calculated_direction = self._azimuth_to_direction(azimuth)

        return {
            "is_valid": calculated_direction == claimed_direction,
            "calculated_azimuth": round(azimuth, 1),
            "calculated_direction": calculated_direction
        }

    def _azimuth_to_direction(self, azimuth: float) -> str:
        """将方位角转换为8方向"""
        if azimuth < 22.5 or azimuth >= 337.5: return "北"
        elif azimuth < 67.5: return "东北"
        elif azimuth < 112.5: return "东"
        elif azimuth < 157.5: return "东南"
        elif azimuth < 202.5: return "南"
        elif azimuth < 247.5: return "西南"
        elif azimuth < 292.5: return "西"
        else: return "西北"
```

```python
class GeoSRBenchEvaluator:
    """GeoSR-Bench评测器"""

    def evaluate_multiple_choice(self, prediction: str, ground_truth: str) -> float:
        """选择题评分：完全匹配得1分"""
        return 1.0 if prediction.strip().upper() == ground_truth.upper() else 0.0

    def evaluate_fill_blank(self, prediction: str, ground_truth: dict) -> float:
        """填空题评分：容忍误差"""
        try:
            pred_value = float(re.search(r'[\d.]+', prediction).group())
            exact = ground_truth['exact_answer']
            tolerance = ground_truth['tolerance']

            if abs(pred_value - exact) <= tolerance:
                return 1.0
            elif abs(pred_value - exact) <= tolerance * 2:
                return 0.5
            return 0.0
        except:
            return 0.0

    def evaluate_reasoning(self, prediction: str, ground_truth: dict) -> float:
        """推理题评分：答案+推理链"""
        # 使用GPT-4评估推理链完整性
        score = 0.0
        if ground_truth['correct_answer'] in prediction:
            score += 0.6  # 答案正确
        # 检查推理步骤
        for step in ground_truth['reasoning_steps']:
            if any(kw in prediction for kw in step.split()[:3]):
                score += 0.4 / len(ground_truth['reasoning_steps'])
        return min(score, 1.0)
```

## 4.6.2 基线实验实现

### 标准KL蒸馏代码

```python
# baselines/standard_kd.py
import torch
import torch.nn as nn
import torch.nn.functional as F
from transformers import AutoModelForCausalLM, AutoTokenizer
from tqdm import tqdm

class StandardKDDistiller:
    """标准KL散度蒸馏"""

    def __init__(self, teacher_model, student_model, tokenizer, temperature=2.0):
        self.teacher = teacher_model
        self.student = student_model
        self.tokenizer = tokenizer
        self.temperature = temperature

        # 冻结教师模型
        for param in self.teacher.parameters():
            param.requires_grad = False
        self.teacher.eval()

    def distillation_loss(self, teacher_logits, student_logits, labels):
        """计算蒸馏损失"""
        # KL散度损失
        teacher_probs = F.softmax(teacher_logits / self.temperature, dim=-1)
        student_log_probs = F.log_softmax(student_logits / self.temperature, dim=-1)

        kl_loss = F.kl_div(student_log_probs, teacher_probs, reduction='batchmean')
        kl_loss = kl_loss * (self.temperature ** 2)

        # 任务损失（交叉熵）
        task_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            labels.view(-1),
            ignore_index=self.tokenizer.pad_token_id
        )

        # 总损失
        alpha = 0.5  # 蒸馏损失权重
        total_loss = alpha * kl_loss + (1 - alpha) * task_loss

        return total_loss, kl_loss, task_loss

    def train_step(self, batch):
        """单步训练"""
        input_ids = batch['input_ids'].to(self.student.device)
        attention_mask = batch['attention_mask'].to(self.student.device)
        labels = batch['labels'].to(self.student.device)

        # 教师模型推理
        with torch.no_grad():
            teacher_outputs = self.teacher(
                input_ids=input_ids,
                attention_mask=attention_mask
            )
            teacher_logits = teacher_outputs.logits

        # 学生模型推理
        student_outputs = self.student(
            input_ids=input_ids,
            attention_mask=attention_mask
        )
        student_logits = student_outputs.logits

        # 计算损失
        total_loss, kl_loss, task_loss = self.distillation_loss(
            teacher_logits, student_logits, labels
        )

        return total_loss, kl_loss, task_loss

    def train_epoch(self, dataloader, optimizer, scheduler):
        """训练一个epoch"""
        self.student.train()
        total_loss = 0

        for batch in tqdm(dataloader, desc="Training"):
            optimizer.zero_grad()

            loss, kl_loss, task_loss = self.train_step(batch)
            loss.backward()

            optimizer.step()
            scheduler.step()

            total_loss += loss.item()

        return total_loss / len(dataloader)
```

### 空间关系感知蒸馏损失（完整实现）

```python
# models/spatial_aware_loss.py
import torch
import torch.nn as nn
import torch.nn.functional as F
import re
from typing import List, Dict, Tuple

class SpatialRelationAwareDistillationLoss(nn.Module):
    """
    空间关系类型感知的蒸馏损失（核心创新）

    核心思想：不同类型的空间关系在蒸馏过程中应该有不同的权重
    - 方向关系（东、西、南、北）是空间认知的基础，权重最高
    - 拓扑关系（包含、相邻）是空间推理的核心，权重较高
    - 度量关系（距离、面积）相对容易学习，权重适中
    """

    def __init__(self, temperature: float = 2.0, use_dynamic_weight: bool = False):
        super().__init__()
        self.temperature = temperature
        self.use_dynamic_weight = use_dynamic_weight

        # 空间关系类型权重（基于空间认知理论设定）
        self.relation_weights = {
            'directional': 1.5,   # 方向关系：空间认知基础
            'topological': 1.3,   # 拓扑关系：空间推理核心
            'metric': 1.0,        # 度量关系：相对容易
            'order': 1.0,         # 序关系：中等难度
            'complex': 1.8,       # 综合推理：最高难度
            'general': 1.0        # 一般文本：标准权重
        }

        # 空间关系词表
        self._init_vocabulary()

    def _init_vocabulary(self):
        """初始化空间关系识别词表"""
        self.direction_words = [
            '东', '西', '南', '北', '东北', '西南', '东南', '西北',
            '左边', '右边', '前面', '后面', '上面', '下面',
            '东侧', '西侧', '南侧', '北侧', '方向', '方位'
        ]
        self.topology_words = [
            '包含', '位于', '相邻', '接壤', '穿过', '流经',
            '内部', '外部', '边界', '交界', '属于', '辖区'
        ]
        self.metric_words = [
            '距离', '远', '近', '公里', '千米', '米', '面积',
            '大小', '长度', '宽度', '半径', '直径'
        ]
        self.order_words = [
            '之前', '之后', '先', '后', '最', '第一', '最后'
        ]

    def identify_relation_type(self, text: str) -> str:
        """
        识别文本中的空间关系类型

        使用规则+词表的方法，按优先级匹配：
        1. 检查数据中的显式标注
        2. 基于词表的关键词匹配
        3. 默认为general
        """
        # 优先级1：方向关系
        for word in self.direction_words:
            if word in text:
                return 'directional'

        # 优先级2：拓扑关系
        for word in self.topology_words:
            if word in text:
                return 'topological'

        # 优先级3：度量关系
        for word in self.metric_words:
            if word in text:
                return 'metric'

        # 优先级4：序关系
        for word in self.order_words:
            if word in text:
                return 'order'

        return 'general'

    def forward(
        self,
        teacher_logits: torch.Tensor,
        student_logits: torch.Tensor,
        input_texts: List[str],
        labels: torch.Tensor,
        spatial_types: List[str] = None
    ) -> Tuple[torch.Tensor, Dict]:
        """
        计算空间关系感知的蒸馏损失

        Args:
            teacher_logits: [batch_size, seq_len, vocab_size]
            student_logits: [batch_size, seq_len, vocab_size]
            input_texts: 输入文本列表
            labels: 标签 [batch_size, seq_len]
            spatial_types: 预定义的空间关系类型（可选）

        Returns:
            total_loss: 总损失
            loss_dict: 各部分损失的字典
        """
        batch_size = teacher_logits.size(0)
        device = teacher_logits.device

        # 1. 计算每个样本的空间关系权重
        if spatial_types is None:
            spatial_types = [self.identify_relation_type(text) for text in input_texts]

        weights = torch.tensor(
            [self.relation_weights[t] for t in spatial_types],
            device=device
        )

        # 2. 计算KL散度（逐样本加权）
        teacher_probs = F.softmax(teacher_logits / self.temperature, dim=-1)
        student_log_probs = F.log_softmax(student_logits / self.temperature, dim=-1)

        # KL散度: sum(p * log(p/q))
        kl_per_sample = torch.sum(
            teacher_probs * (torch.log(teacher_probs + 1e-10) - student_log_probs),
            dim=-1
        ).mean(dim=-1)  # [batch_size]

        # 加权KL损失
        weighted_kl_loss = (weights * kl_per_sample).mean()
        weighted_kl_loss = weighted_kl_loss * (self.temperature ** 2)

        # 3. 任务损失（交叉熵）
        task_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            labels.view(-1),
            ignore_index=-100,
            reduction='mean'
        )

        # 4. 总损失（加权组合）
        alpha = 0.5
        total_loss = alpha * weighted_kl_loss + (1 - alpha) * task_loss

        # 5. 记录各类型损失
        loss_dict = {
            'total_loss': total_loss.item(),
            'weighted_kl_loss': weighted_kl_loss.item(),
            'task_loss': task_loss.item(),
            'spatial_type_counts': {t: spatial_types.count(t) for t in set(spatial_types)}
        }

        return total_loss, loss_dict


class GeoKDDistiller:
    """完整的地理知识蒸馏框架"""

    def __init__(
        self,
        teacher_model,
        student_model,
        tokenizer,
        temperature: float = 2.0,
        use_reverse_kl: bool = True
    ):
        self.teacher = teacher_model
        self.student = student_model
        self.tokenizer = tokenizer
        self.use_reverse_kl = use_reverse_kl

        # 初始化损失函数
        self.spatial_loss = SpatialRelationAwareDistillationLoss(temperature)

        # 冻结教师模型
        for param in self.teacher.parameters():
            param.requires_grad = False
        self.teacher.eval()

    def compute_reverse_kl(self, teacher_logits, student_logits):
        """计算逆向KL散度（借鉴DeepSeek-R1）"""
        student_probs = F.softmax(student_logits, dim=-1)
        teacher_probs = F.softmax(teacher_logits, dim=-1)

        reverse_kl = torch.sum(
            student_probs * (torch.log(student_probs + 1e-10) - torch.log(teacher_probs + 1e-10)),
            dim=-1
        ).mean()

        return reverse_kl

    def train_step(self, batch):
        """单步训练"""
        input_ids = batch['input_ids']
        attention_mask = batch['attention_mask']
        labels = batch['labels']
        input_texts = batch.get('input_texts', [''] * len(input_ids))
        spatial_types = batch.get('spatial_relation_type', None)

        # 教师模型推理
        with torch.no_grad():
            teacher_outputs = self.teacher(
                input_ids=input_ids,
                attention_mask=attention_mask
            )
            teacher_logits = teacher_outputs.logits

        # 学生模型推理
        student_outputs = self.student(
            input_ids=input_ids,
            attention_mask=attention_mask
        )
        student_logits = student_outputs.logits

        # 计算空间关系感知蒸馏损失
        total_loss, loss_dict = self.spatial_loss(
            teacher_logits, student_logits, input_texts, labels, spatial_types
        )

        # 添加逆向KL损失（可选）
        if self.use_reverse_kl:
            reverse_kl = self.compute_reverse_kl(teacher_logits, student_logits)
            total_loss = total_loss + 0.3 * reverse_kl
            loss_dict['reverse_kl'] = reverse_kl.item()

        return total_loss, loss_dict
```

```python
    train_dataset = GeoSRDataset("./data/geosr_chain/train.jsonl", tokenizer)
    val_dataset = GeoSRDataset("./data/geosr_chain/val.jsonl", tokenizer)

    train_loader = DataLoader(train_dataset, batch_size=8, shuffle=True)
    val_loader = DataLoader(val_dataset, batch_size=8)

    # 5. 优化器配置（借鉴K2论文）
    optimizer = torch.optim.AdamW(
        student.parameters(),
        lr=1e-4,
        weight_decay=0.01
    )

    num_epochs = 3
    num_training_steps = len(train_loader) * num_epochs
    scheduler = get_linear_schedule_with_warmup(
        optimizer,
        num_warmup_steps=int(0.1 * num_training_steps),
        num_training_steps=num_training_steps
    )

    # 6. 训练循环
    best_val_loss = float('inf')

    for epoch in range(num_epochs):
        print(f"\n=== Epoch {epoch+1}/{num_epochs} ===")

        # 训练
        student.train()
        train_loss = 0
        for batch in tqdm(train_loader, desc="Training"):
            optimizer.zero_grad()
            loss, loss_dict = distiller.train_step(batch)
            loss.backward()
            optimizer.step()
            scheduler.step()
            train_loss += loss.item()

        avg_train_loss = train_loss / len(train_loader)
        print(f"Train Loss: {avg_train_loss:.4f}")

        # 验证
        student.eval()
        val_loss = 0
        with torch.no_grad():
            for batch in tqdm(val_loader, desc="Validation"):
                loss, _ = distiller.train_step(batch)
                val_loss += loss.item()

        avg_val_loss = val_loss / len(val_loader)
        print(f"Val Loss: {avg_val_loss:.4f}")

        # 保存最佳模型
        if avg_val_loss < best_val_loss:
            best_val_loss = avg_val_loss
            student.save_pretrained(f"./checkpoints/geokd_sr_best")
            print(f"保存最佳模型 (val_loss: {best_val_loss:.4f})")

    print("\n训练完成!")

if __name__ == "__main__":
    main()
```
