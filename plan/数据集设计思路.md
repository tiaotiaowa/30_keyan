# GeoKD-SR 基于OSM真实数据的地理空间推理数据集设计方案

## Context

当前GeoKD-SR项目使用的11,770条数据由GLM API生成，存在事实性错误风险（幻觉实体、错误坐标、错误空间关系）。为确保研究数据的科学性和可重复性，决定基于OSM真实数据和行政区划边界数据，使用PostGIS空间数据库精确计算空间关系，完全替代现有数据集。

**目标期刊**: ISPRS IJGI 特刊 "LLM4GIS"
**数据规模**: 10,000条，按8:1:1划分（训练8000/验证1000/测试1000）
**语言**: 中英双语
**工作目录**: D:\gis_data

## 二、实体系统设计

### 2.1 统一实体格式（精简版）

```json
{
  "entity_id": "admin_city_420100",
  "entity_type": "administrative",
  "entity_subtype": "city",
  "name_zh": "武汉市",
  "name_en": "Wuhan",
  "geometry_type": "Polygon",
  "centroid": [114.3055, 30.5930],
  "source": "geoatlas",
  "properties": { "adcode": 420100, "level": "city" }
}
```

**字段说明**:
- entity_id: 唯一标识 (格式: `{type}_{subtype}_{id}`)
- entity_type: 11种之一(administrative/terrain/water_body/waterway/landmark/facility/transport/area/road/railway)
- entity_subtype: 具体子类型(province/city/peak/lake/river/attraction/university/station/park等)
- name_zh/name_en: 中英文名称
- geometry_type: Point/Polygon/Line
- centroid: [经度, 纬度] (WGS84)
- source: geoatlas/osm
- properties: 类型特有属性(行政区划有adcode/level，山峰有elevation等)

### 2.2 实体类型体系

| 实体类型 | entity_type | 子类型 | 几何 | 来源 | 数量 |
|---------|-------------|--------|------|------|------|
| **行政区划** | administrative | province | Polygon | geoatlas | 35 |
| **行政区划** | administrative | city/district | Polygon | geoatlas | 475 |
| **山峰/山脉** | terrain | peak | Point | OSM natural | ~1万 |
| **湖泊/水库** | water_body | lake/reservoir | Polygon | OSM water_a | ~3千 |
| **河流** | waterway | river/canal | Line | OSM waterways | ~2万 |
| **景点** | landmark | attraction | Point | OSM pois | ~5万 |
| **公共设施** | facility | university/hospital | Point | OSM pois | ~3万 |
| **交通设施** | transport | station/airport | Point | OSM transport | ~2万 |
| **森林/公园** | area | forest/park | Polygon | OSM landuse_a | ~1万 |
| **主要道路** | road | national_motorway/trunk | Line | OSM roads | ~3万 |
| **铁路** | railway | rail | Line | OSM railways | ~5千 |

### 3.2 方向推理 (Directional) — 2,500条

**计算方法**: 统一使用点-点（质心）计算

```
PostGIS: ST_Azimuth(ST_Centroid(A), ST_Centroid(B))
弧度 → 角度 → 8方位映射
北(337.5-22.5°), 东北(22.5-67.5°), 东(67.5-112.5°), 东南(112.5-157.5°)
南(157.5-202.5°), 西南(202.5-247.5°), 西(247.5-292.5°), 西北(292.5-337.5°)
```

**实体对组合**:

| 编号 | 实体A (Point) | 实体B (Point) | 可用对数 | 采样数 |
|------|-------------|-------------|---------|--------|
| D1 | province.centroid | city.centroid | 35×475=16,625 | 500 |
| D2 | city.centroid | city.centroid (跨省) | 475×474/2=112,575 | 700 |
| D3 | city.centroid | peak.point | 475×千级 | 500 |
| D4 | city.centroid | lake.centroid | 475×千级 | 400 |
| D5 | city.centroid | attraction.point | 475×万级 | 400 |

**难度定义**:
- Easy(30%=750): 同省实体对，方向明确
- Medium(50%=1,250): 跨省实体对，中等距离
- Hard(20%=500): 方位接近边界值(如东北偏北vs正北)、需排除干扰项

## 五、难度体系设计

### 5.1 难度定义

**方向推理难度**:
- Easy: 同省实体对，距离<200km，方向明确无歧义
- Medium: 跨省实体对，距离200-800km
- Hard: 方位接近边界值、需排除干扰、超远距离(>800km)

**拓扑推理难度**:
- Easy: 明确的包含/相邻关系(如省包含市、省际相邻)
- Medium: 中等粒度(市-市相邻、河流穿越省份)
- Hard: 近边界判断、负样本辨别(接近但不包含/不相邻)、多步推理

**距离推理难度**:
- Easy: 单对精确距离(给出两个实体问距离)
- Medium: 两对距离比较(哪个更近？近多少？)
- Hard: 多实体排序、距离区间推理

**复合推理难度**:
- Easy: 两重组合(方向+距离)
- Medium: 两重组合(方向+拓扑或距离+拓扑)
- Hard: 三重组合、多步链式推理

### 5.2 难度分配

| 难度 | 占比 | 条数 |
|------|------|------|
| Easy | 30% | 3,000 |
| Medium | 50% | 5,000 |
| Hard | 20% | 2,000 |

### 5.3 难度分数计算 (difficulty_score) — 双轨难度体系

**核心原则**:
1. **LLM没有计算器** — 即使给了坐标，方向/距离也不是"简单算术"，模型需要数值推理
2. **复合 > 单一拓扑** — 多维度整合是更高阶的认知负荷
3. **拓扑是所有模式中最难的维度** — 需要完整几何推理，质心坐标帮助有限
4. **带坐标 vs 不带坐标** — 两种模式下所有类型都有难度梯度

#### 5.3.1 各维度固有难度

| 推理维度 | 带坐标难度 | 不带坐标难度 | 原因 |
|---------|-----------|------------|------|
| 方向推理 | 0.25 | 0.50 | 带坐标:需比较Δlon/Δlat大小比值映射8方位；不带:需空间直觉 |
| 距离推理 | 0.30 | 0.55 | 带坐标:需从经纬度差估算公里数(含球面修正)；不带:需距离知识 |
| 拓扑.contains | 0.30 | 0.60 | 质心坐标无法判断包含关系，需完整polygon边界 |
| 拓扑.within | 0.30 | 0.60 | 同上，视角互换 |
| 拓扑.disjoint | 0.40 | 0.65 | 需确认完全无交集 |
| 拓扑.touches | 0.50 | 0.70 | 需判断边界共享 |
| 拓扑.crosses | 0.65 | 0.80 | 线穿面，最高几何复杂度 |

#### 5.3.2 复合难度 = max(各分量) + 整合奖励

| 复合类型 | 计算公式(带坐标) | 结果范围(带坐标) | 计算公式(不带坐标) | 结果范围(不带坐标) |
|---------|----------------|-----------------|------------------|------------------|
| C1(方向+距离) | max(0.25,0.30)+0.15 | 0.45 | max(0.50,0.55)+0.15 | 0.70 |
| C2(方向+拓扑) | max(0.25,topo)+0.20 | 0.50~0.85 | max(0.50,topo)+0.20 | 0.70~0.90 |
| C3(距离+拓扑) | max(0.30,topo)+0.20 | 0.50~0.85 | max(0.55,topo)+0.20 | 0.75~0.90 |
| C4(三重) | max(0.30,topo)+0.30 | 0.60~0.95 | max(0.55,topo)+0.30 | 0.85~1.00 |

**关键排序验证(带坐标)**:
```
directional(0.25) < metric(0.30) ≤ contains/within(0.30) < disjoint(0.40)
< C1(0.45) < touches(0.50) < C2/C3(0.50~0.85) < crosses(0.65) < C4(0.60~0.95)
```
✓ C4(0.60~0.95) > 所有单一拓扑 ✓ C2/C3(0.50~0.85) > 大多数单一拓扑

#### 5.3.3 调节因子（适用于所有类型）

| 因子 | 权重 | 适用类型 | 评分规则 |
|------|------|---------|---------|
| 方向边界接近度 | 0.30 | directional/composite含方向 | `1-min(az%45,45-az%45)/22.5`→0~1 |
| 距离估算难度 | 0.25 | metric/composite含距离 | <10km→0.8, 10-100→0.2, 100-500→0.4, >500→0.6 |
| 几何交互复杂度 | 0.25 | topological/composite含拓扑 | Pt-Pt=0.1, Pt-Pg=0.3, Pg-Pg=0.5, Ln-Pg=0.9 |
| 空间尺度微调 | 0.20 | 全部 | <50km→+0.1, >1000km→+0.1, 其他→0 |

#### 5.3.4 计算公式

```python
# 基准难度表
BASE = {
    # (relation_type, has_coords): base_score
    ("directional",  True):  0.25,  ("directional",  False): 0.50,
    ("metric",       True):  0.30,  ("metric",       False): 0.55,
    ("contains",     True):  0.30,  ("contains",     False): 0.60,
    ("within",       True):  0.30,  ("within",       False): 0.60,
    ("disjoint",     True):  0.40,  ("disjoint",     False): 0.65,
    ("touches",      True):  0.50,  ("touches",      False): 0.70,
    ("crosses",      True):  0.65,  ("crosses",      False): 0.80,
}
INTEGRATION_BONUS = {
    "comp_dir_dist": 0.15,   # 两算术整合
    "comp_dir_topo": 0.20,   # 算术+几何跨域整合
    "comp_dist_topo":0.20,   # 算术+几何跨域整合
    "comp_all":      0.30,   # 三维全整合
}

def compute_difficulty(pair, has_coords=True):
    rel = pair['relation_key']

    if rel.startswith("comp_"):
        # 复合: max(各分量基准) + 整合奖励
        components = get_component_bases(pair, has_coords)
        base = max(components) + INTEGRATION_BONUS[rel]
    else:
        base = BASE[(rel, has_coords)]

    # 调节因子(0~0.15范围微调)
    adj = 0.0
    if has_direction_component(pair):
        az = pair['spatial_result'].get('azimuth_deg', 45)
        boundary_prox = 1 - min(az % 45, 45 - az % 45) / 22.5
        adj += 0.30 * boundary_prox * 0.15  # 最大贡献0.045

    if has_distance_component(pair):
        d = pair['spatial_result'].get('distance_km', 200)
        if d < 10: ds = 0.8
        elif d < 100: ds = 0.2
        elif d < 500: ds = 0.4
        else: ds = 0.6
        adj += 0.25 * ds * 0.15

    if has_topology_component(pair):
        geo = tuple(sorted([pair['entity_a']['geometry_type'],
                            pair['entity_b']['geometry_type']]))
        geo_s = {("Point","Point"):0.1,("Point","Polygon"):0.3,
                 ("Polygon","Polygon"):0.5,("Line","Polygon"):0.9}
        adj += 0.25 * geo_s.get(geo, 0.3) * 0.15

    difficulty_score = round(min(base + adj, 1.0), 2)
    if difficulty_score < 0.33: difficulty = "easy"
    elif difficulty_score < 0.66: difficulty = "medium"
    else: difficulty = "hard"
    return difficulty, difficulty_score
```

#### 5.3.5 难度分布控制 — 目标30%/50%/20%

**按类型目标分配（总计 Easy:3000, Medium:5000, Hard:2000）**:

| 类型 | 总条数 | Easy | Medium | Hard | 说明 |
|------|--------|------|--------|------|------|
| 方向 | 2,500 | 1,100(44%) | 1,200(48%) | 200(8%) | 带坐标时方向推理总体偏易 |
| 距离 | 2,500 | 900(36%) | 1,300(52%) | 300(12%) | 距离估算难度略高于方向 |
| 拓扑 | 2,500 | 650(26%) | 1,200(48%) | 650(26%) | 子类型间差异大 |
| 复合 | 2,500 | 350(14%) | 1,300(52%) | 850(34%) | 复合难度最高 |
| **合计** | **10,000** | **3,000(30%)** | **5,000(50%)** | **2,000(20%)** | ✓ |

**实现方式**: 在每个类型内，按computed difficulty_score排序，取百分位截断：
- 在类型i内按score排序，取前target_Easy[i]条为Easy，接下来target_Medium[i]条为Medium，其余为Hard
- 这样保证了类型内难度排序合理 + 总体30/50/20精确达标

#### 5.3.6 数据划分均匀性保证

**分层划分策略**: 按 `relation_type × difficulty_level` 共 4×3=12 个层进行分层采样

| 均匀性维度 | 保证方法 |
|-----------|---------|
| **难度均匀** | 每层按 8:1:1 分配到 train/dev/test，各split难度分布一致 |
| **关系类型均匀** | 每个relation_type在train/dev/test中比例一致 |
| **实体对均匀** | 实体对互斥(同一对只在一个split) + 各split中实体类型分布统计验证 |
| **拓扑子类型均匀** | 5种拓扑子类型在各split中等比例分配 |
| **验证标准** | 各split间类型分布TVD<0.05, 难度分布TVD<0.05 |

```python
def stratified_split(items, train_ratio=0.8, dev_ratio=0.1, test_ratio=0.1):
    """分层划分: 按 (relation_type, difficulty) 分层"""
    from collections import defaultdict

    # 按 (relation_type, difficulty) 分组
    groups = defaultdict(list)
    for item in items:
        key = (item['spatial_relation_type'], item['difficulty'])
        groups[key].append(item)

    train, dev, test = [], [], []
    for key, group in groups.items():
        random.shuffle(group)
        n = len(group)
        n_train = int(n * train_ratio)
        n_dev = int(n * dev_ratio)
        train.extend(group[:n_train])
        dev.extend(group[n_train:n_train+n_dev])
        test.extend(group[n_train+n_dev:])

    return train, dev, test
```

## 七、数据划分策略

### 7.1 划分原则

**三重均匀性**: 难度均匀 × 关系类型均匀 × 实体对均匀，通过 `relation_type × difficulty_level` 分层采样实现。

### 7.2 分层划分方案

| 层级 | 分层键 | 层数 | 说明 |
|------|--------|------|------|
| L1 | relation_type | 4 | 方向/拓扑/距离/复合 |
| L2 | difficulty_level | 3 | Easy/Medium/Hard |
| L3 | topology_subtype(仅拓扑) | 5 | contains/within/touches/crosses/disjoint |
| L4 | composite_type(仅复合) | 4 | C1/C2/C3/C4 |

**共 12层(relation×difficulty) + 拓扑5子类型 + 复合4子类型 = 精细分层**

### 7.3 分配比例 — 11种子类型严格8:1:1

**每种子类型独立8:1:1划分，确保所有关系在各split中完全均匀**:

| 子类型 | 总条数 | train(80%) | dev(10%) | test(10%) |
|--------|--------|-----------|---------|----------|
| 方向(directional) | 2,500 | 2,000 | 250 | 250 |
| 距离(metric) | 2,500 | 2,000 | 250 | 250 |
| 拓扑.contains | 500 | 400 | 50 | 50 |
| 拓扑.within | 500 | 400 | 50 | 50 |
| 拓扑.touches | 500 | 400 | 50 | 50 |
| 拓扑.crosses | 500 | 400 | 50 | 50 |
| 拓扑.disjoint | 500 | 400 | 50 | 50 |
| 复合.C1(方向+距离) | 875 | 700 | 87 | 88 |
| 复合.C2(方向+拓扑) | 500 | 400 | 50 | 50 |
| 复合.C3(距离+拓扑) | 500 | 400 | 50 | 50 |
| 复合.C4(三重) | 625 | 500 | 62 | 63 |
| **合计** | **10,000** | **8,000** | **1,000** | **1,000** |

**实现**: 在每种子类型内独立shuffle后按8:1:1切分，无需跨类型调整

### 7.4 实体对互斥 + 泄露分析

**实体对互斥**: 同一对实体（如湖北-武汉）的所有关系类型都在同一个split

**泄露分析（应对审稿质疑）**:
1. 在test集中标记: **seen_pair**(两实体分别在train不同对中出现过) vs **unseen_pair**(至少一实体在train中从未出现)
2. 分别计算准确率，若差异<5% → 证明模型学的是推理模式而非实体记忆

### 7.5 均匀性验证标准

| 指标 | 验证方法 | 通过标准 |
|------|---------|---------|
| 关系类型分布 | 各split的4类比例 | TVD < 0.05 |
| 拓扑子类型分布 | 各split的5子类比例 | TVD < 0.05 |
| 难度分布 | 各split的E/M/H比例 | TVD < 0.05 |
| 实体类型覆盖 | 各split中11种实体类型出现率 | 覆盖率 > 90% |

## 十、实施步骤概览

### Step 1: 环境搭建 ✅ (已完成)
- PostgreSQL + PostGIS 已安装在D盘
- 需创建数据库和空间扩展
- 解压OSM Shapefile数据

### Step 2: 数据导入
- geoatlas GeoJSON导入PostGIS
- OSM Shapefile批量导入(32省)
- 统一坐标系WGS84

### Step 3: 实体提取与分类
- 按筛选条件提取各类实体
- 构建统一实体库
- 生成中英文实体名映射

### Step 4: 空间关系计算
- 方向: ST_Azimuth批量计算
- 拓扑: ST_Contains/ST_Within/ST_Touches/ST_Crosses/ST_Disjoint
- 距离: ST_DistanceSphere批量计算
- 复合: 组合计算

### Step 5: 实体对构建
- 基于空间关系计算结果，构建10,000条实体对
- 按比例分配4种空间关系类型(各2,500条)
- 标注参考实体(directional/contains/within/crosses)

### Step 5.5: LLM QA生成(GLM-4.7 API)
- **输入**: 实体对JSON(含实体名、坐标、关系类型、空间计算结果、参考实体)
- **输出**: 自然语言QA + 结构化参考答案
- **Prompt策略**:
  - 提供实体对的完整spatial_result作为事实依据
  - 要求生成多样化问题风格(判断题/选择题/填空题/推理题)
  - 同时输出:
    1. question_zh / question_en (自然语言问题)
    2. answer_zh / answer_en (自然语言答案)
    3. answer_structured (结构化参考答案, 如 {"direction": "northeast", "distance_km": 296})
    4. reasoning_chain (5步推理链)
    5. difficulty (easy/medium/hard) + difficulty_score (0-1)
    6. spatial_tokens / entity_to_token

### Step 6: 质量验证
- PostGIS反向验证空间关系
- 实体/坐标/关系一致性检查
- LLM生成的答案与结构化答案交叉验证
- 分布统计验证

### Step 7: 数据划分与导出
- 实体对互斥划分
- 分层采样(train/dev/test)
- 导出最终JSONL文件
