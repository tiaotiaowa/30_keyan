#### 各方法详细设计

**B1: Direct-SFT（对照组）**
```python
# 直接监督微调，无蒸馏
def direct_sft_loss(student_logits, labels):
    return F.cross_entropy(student_logits, labels)
```

**B2: Standard-KD（经典基线）**
```python
# Hinton 2015 经典KL蒸馏
def standard_kd_loss(teacher_logits, student_logits, labels, alpha=0.5, T=2.0):
    # 软标签蒸馏
    soft_loss = F.kl_div(
        F.log_softmax(student_logits / T, dim=-1),
        F.softmax(teacher_logits / T, dim=-1),
        reduction='batchmean'
    ) * (T ** 2)
    # 硬标签损失
    hard_loss = F.cross_entropy(student_logits, labels)
    return alpha * soft_loss + (1 - alpha) * hard_loss
```

**B3: MiniLLM（逆向KL蒸馏）⭐ 关键对比基线**
```python
# Microsoft ICLR 2024: 逆向KL蒸馏
# 参考: Gu et al. "MiniLLM: Knowledge Distillation of Large Language Models"
def reverse_kl_loss(student_logits, teacher_logits, temperature=1.0):
    """
    逆向KL散度：KL(p_student || p_teacher)

    优点（相比Forward KL）：
    - 避免mode-covering问题（学生不会过估计低概率区域）
    - 聚焦于教师的主要模式
    - 更适合生成式任务
    - 减少exposure bias

    参考: https://github.com/microsoft/LMOps/tree/main/minillm
    """
    student_probs = F.softmax(student_logits / temperature, dim=-1)
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)

    # Reverse KL: 学生分布在前，教师分布在后
    rkl = torch.sum(
        student_probs * (
            torch.log(student_probs + 1e-10) -
            torch.log(teacher_probs + 1e-10)
        ),
        dim=-1
    )
    return rkl.mean()

def minillm_loss(teacher_logits, student_logits, labels, alpha=0.7):
    """MiniLLM完整损失函数"""
    # 逆向KL蒸馏损失
    rkl_loss = reverse_kl_loss(student_logits, teacher_logits)

    # 任务损失
    task_loss = F.cross_entropy(
        student_logits.view(-1, student_logits.size(-1)),
        labels.view(-1),
        ignore_index=-100
    )

    return alpha * rkl_loss + (1 - alpha) * task_loss
```

**为什么MiniLLM是关键基线**:
- 2024年Microsoft系统性研究，ICLR接收
- 证明逆向KL在生成任务上优于正向KL
- 当前大模型蒸馏的最佳实践之一

**B4: CoT-Distill（思维链蒸馏）**
```python
# 教师生成思维链供学生学习
# 参考: Shridhar et al. "Distilling Reasoning Capabilities into Smaller Language Models" (ACL 2023)
class CoTDistillLoss(nn.Module):
    """
    思维链蒸馏

    核心思想: 教师生成推理链，学生学习完整推理过程
    区别于Standard-KD: 不仅学习答案分布，还学习推理步骤
    """
    def __init__(self, alpha=0.5):
        super().__init__()
        self.alpha = alpha

    def forward(self, student_logits, cot_labels, answer_labels):
        """
        Args:
            student_logits: 学生模型输出
            cot_labels: 思维链部分的标签
            answer_labels: 最终答案的标签
        """
        # 思维链损失: 学习推理过程
        cot_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            cot_labels.view(-1),
            ignore_index=-100
        )

        # 答案损失: 学习最终答案
        answer_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            answer_labels.view(-1),
            ignore_index=-100
        )

        return self.alpha * cot_loss + (1 - self.alpha) * answer_loss
```

#### 基线对比维度

| 对比维度 | 基线对比 | 验证的假设 |
|---------|---------|-----------|
| 蒸馏 vs 无蒸馏 | B1 vs B2/B3/B4 | 知识蒸馏能提升性能 |
| Forward KL vs Reverse KL | **B2 vs B3** | 逆向KL更适合生成任务 |
| 答案蒸馏 vs 推理蒸馏 | B2/B3 vs B4 | 思维链提升推理能力 |
| 通用方法 vs 空间感知 | B2/B3/B4 vs GeoKD-SR | 领域知识增强蒸馏 |

#### 为什么删除TinyBERT-style?

1. **架构不匹配**: TinyBERT针对BERT(编码器)，不适用于Qwen(解码器)
2. **显存占用高**: 需要对齐隐藏层，显存需求~14GB vs 9GB
3. **学术说服力弱**: 没有针对生成式LLM的特征蒸馏经典工作
4. **替代方案**: MiniLLM(逆向KL)是更合适的LLM时代基线

### 阶段3：核心创新实现（3-5天）

**目标**: 实现GeoKD-SR空间关系感知蒸馏

#### 核心创新：空间关系类型感知的蒸馏损失

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

class SpatialAwareDistillationLoss(nn.Module):
    """
    空间关系类型感知的蒸馏损失（核心创新）

    核心思想：不同类型的空间关系在蒸馏过程中应该有不同的权重
    - 方向关系（东、西、南、北）是空间认知的基础，权重最高
    - 拓扑关系（包含、相邻）是空间推理的核心，权重较高
    - 度量关系（距离、面积）相对容易学习，权重适中
    """

    def __init__(self, temperature=2.0):
        super().__init__()
        self.temperature = temperature

        # 空间关系类型权重（基于空间认知理论设定）
        self.weights = {
            "directional": 1.5,  # 方向关系：空间认知基础
            "topological": 1.3,  # 拓扑关系：空间推理核心
            "metric": 1.0,       # 度量关系：相对容易
            "composite": 1.8     # 组合推理：最高难度
        }

        # 空间关系识别词表
        self.direction_words = ['东', '西', '南', '北', '东北', '西南', '东南', '西北']
        self.topology_words = ['包含', '位于', '相邻', '接壤', '穿过', '流经', '内部']
        self.metric_words = ['距离', '远', '近', '公里', '千米', '米', '面积']

    def identify_relation_type(self, text):
        """识别文本中的空间关系类型"""
        for word in self.direction_words:
            if word in text:
                return 'directional'
        for word in self.topology_words:
            if word in text:
                return 'topological'
        for word in self.metric_words:
            if word in text:
                return 'metric'
        return 'composite'

    def forward(self, teacher_logits, student_logits, input_texts, labels):
        """
        计算空间关系感知的蒸馏损失

        Args:
            teacher_logits: [batch_size, seq_len, vocab_size]
            student_logits: [batch_size, seq_len, vocab_size]
            input_texts: 输入文本列表
            labels: 标签
        """
        batch_size = teacher_logits.size(0)
        device = teacher_logits.device

        # 识别每个样本的空间关系类型并获取权重
        weights = torch.tensor([
            self.weights[self.identify_relation_type(text)]
            for text in input_texts
        ], device=device)

        # 计算KL散度
        teacher_probs = F.softmax(teacher_logits / self.temperature, dim=-1)
        student_log_probs = F.log_softmax(student_logits / self.temperature, dim=-1)

        kl_per_sample = torch.sum(
            teacher_probs * (torch.log(teacher_probs + 1e-10) - student_log_probs),
            dim=-1
        ).mean(dim=-1)

        # 加权KL损失
        weighted_kl_loss = (weights * kl_per_sample).mean() * (self.temperature ** 2)

        # 任务损失
        task_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            labels.view(-1),
            ignore_index=-100
        )

        # 总损失
        alpha = 0.5
        total_loss = alpha * weighted_kl_loss + (1 - alpha) * task_loss

        return total_loss
```

**权重设计原理**：
- 方向关系(1.5)：8方向判断是空间推理核心能力
- 拓扑关系(1.3)：包含/相邻等关系较重要
- 度量关系(1.0)：距离计算相对简单
- 组合推理(1.8)：多步推理最能体现能力差异

| 答案蒸馏 vs 推理蒸馏 | B2/B3 vs B4 | 验证思维链价值 |
| 通用方法 vs 空间感知 | B2/B3/B4 vs GeoKD-SR | 验证领域创新 |

### 9.2 与原方案对比

| 维度 | 原方案(5基线) | 新方案(4基线) |
|------|-------------|--------------|
| 基线数量 | 5个 | 4个 |
| 经典方法 | Standard-KD (2015) | Standard-KD (2015) |
| LLM时代方法 | TinyBERT(2020) + CoT(2024) + DeepSeek(2025) | MiniLLM(2024) + CoT(2023) |
| 对比清晰度 | 混乱 | 清晰 |
| 学术引用 | 部分不明确 | 明确可靠 |
| 实现复杂度 | 高(TinyBERT特征蒸馏复杂) | 中等(都是响应蒸馏) |
| 显存需求 | 最高14GB | 最高10GB |

### 9.3 基线选择理由总结

| 基线 | 选择理由 |
|------|---------|
| B1 Direct-SFT | 必须的对照组，验证蒸馏本身的有效性 |
| B2 Standard-KD | 经典基线，所有知识蒸馏论文都对比 |
| B3 MiniLLM | **关键**: 2024年Microsoft证明逆向KL优于正向KL，ICLR接收 |
| B4 CoT-Distill | 推理能力蒸馏的代表方法，ACL Findings |

### 9.4 预期学术贡献

1. **方法贡献**: 空间关系类型感知的知识蒸馏方法
2. **实证贡献**: 证明逆向KL在空间推理任务上的优势
3. **数据贡献**: 构建高质量的空间推理评测基准
4. **应用价值**: 为地理信息领域的知识蒸馏提供参考
