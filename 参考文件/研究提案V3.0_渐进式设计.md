## 二、Phase 1：MVP版本（当前重点）

### 2.1 研究定位（简化版）

**题目**：面向地理空间关系推理的知识蒸馏方法研究

**核心问题**：如何通过知识蒸馏将大模型的地理空间关系推理能力高效迁移到轻量化模型？

**核心创新（仅1个）**：**空间关系类型感知的蒸馏损失**

### 2.2 技术架构（最小可行）

```
┌─────────────────────────────────────────────────────────────┐
│                  MVP版技术架构（极简设计）                   │
└─────────────────────────────────────────────────────────────┘

输入：地理空间问题（文本）
  ↓
教师模型：Qwen2.5-7B-Instruct
  ↓ 蒸馏
学生模型：Qwen2.5-1.5B（稠密）
  ↓
核心创新：空间关系类型感知蒸馏损失
  - 方向关系损失（权重1.5）
  - 拓扑关系损失（权重1.3）
  - 度量关系损失（权重1.0）
  ↓
输出：地理空间答案
```

### 2.3 实验设计（极简版）

| 实验组 | 描述 | 目的 |
|--------|------|------|
| **基线** | 标准KL散度蒸馏 | 对照组 |
| **实验组** | 空间关系感知蒸馏 | 验证核心创新 |
| **消融1** | 去除方向关系权重 | 分析贡献 |
| **消融2** | 去除拓扑关系权重 | 分析贡献 |

### 2.4 数据集设计（精简版）

| 数据类型 | 规模 | 来源 |
|---------|------|------|
| 训练数据 | 50k | GeoQA + 合成数据 |
| 验证数据 | 5k | 人工标注 |
| 测试数据 | 1k | GeoSR-Bench |

### 2.5 评估指标

| 指标 | 目标 |
|------|------|
| 答案准确率 | 教师模型80%+ |
| 空间关系F1 | 教师模型85%+ |
| 性能提升（vs基线） | 3%+ |
| 推理速度 | 教师模型3x+ |

### 2.6 时间线（Phase 1）

| 月份 | 任务 | 产出 |
|------|------|------|
| **3月** | 基线实验 + 数据V1 | 基线报告 + 50k数据 |
| **4月** | 核心创新实现 | 空间关系感知损失 |
| **5月** | 消融实验 + 优化 | 实验结果 |
| **6月** | 论文撰写 | 论文初稿 |
| **7月** | 投稿 | **IJGI投稿** |

### 2.7 MVP成功标准

```
✅ 必须达成（可发表）：
- 学生模型达到教师模型80%+性能
- 空间关系感知损失提升3%+
- GeoSR-Chain数据集（50k）
- GeoSR-Bench评测基准（1k）

⚠️ 加分项：
- 提升5%+
- 更大规模数据集
```

## 三、Phase 2：扩展版本（投稿后）

### 3.1 扩展方向

| 扩展内容 | 描述 | 创新贡献 |
|---------|------|---------|
| **多教师消融** | 对比通用vs领域教师 | 验证领域预训练价值 |
| **规则式幻觉检测** | 简单的地理事实验证 | 抑制幻觉迁移 |
| **Adapter专家化** | 为不同空间关系添加Adapter | 轻量专业化 |

### 3.2 技术升级

```
MVP架构 + 以下扩展：

1. 多教师对比
   - 教师A：Qwen2.5-7B（通用）
   - 教师B：K2-7B（领域，如可获取）
   - 对比实验

2. 规则式验证
   - 方向验证：检查方向词与坐标一致性
   - 距离验证：检查距离描述与计算一致性
   - 实体验证：检查地理实体是否存在

3. Adapter专家化
   - 方向Adapter：专门处理方向推理
   - 拓扑Adapter：专门处理拓扑推理
   - 距离Adapter：专门处理距离推理
```

### 3.3 Phase 2时间线

| 月份 | 任务 | 产出 |
|------|------|------|
| **8-9月** | 多教师消融实验 | 消融报告 |
| **10月** | 规则式幻觉检测 | 幻觉抑制效果 |
| **11月** | Adapter专家化 | 优化模型 |
| **12月** | 论文撰写 + 投稿 | **CCF-B会议投稿** |

## 五、风险控制与保底方案

### 5.1 分阶段风险控制

| 阶段 | 风险 | 应对策略 |
|------|------|---------|
| Phase 1 | 核心创新效果不显著 | 降低目标至2%提升，仍可发表 |
| Phase 1 | 数据质量问题 | 使用现有GeoQA数据 |
| Phase 2 | 多教师效果不明显 | 简化为单教师分析 |
| Phase 3 | MoE训练不稳定 | 使用成熟框架 |

### 5.2 保底方案（最坏情况）

即使Phase 1只有基线效果，仍可发表：
```
保底论文内容：
1. 地理空间推理蒸馏的实证研究
2. GeoSR-Chain数据集（首个空间推理蒸馏数据）
3. GeoSR-Bench评测基准
4. 挑战与分析

学术价值：数据集贡献 + 实证分析
```

## 七、核心优势总结

### 7.1 渐进式设计的优势

| 优势 | 说明 |
|------|------|
| **风险可控** | MVP确保可发表，扩展作为加分项 |
| **快速产出** | 4个月完成MVP投稿 |
| **持续迭代** | 每阶段都有明确产出 |
| **资源高效** | 优先投入核心创新 |

### 7.2 与原方案对比

| 维度 | 原方案 | 渐进式方案 |
|------|--------|-----------|
| 创新点 | 3个 | 1个核心（Phase 1） |
| 架构复杂度 | MoE + Geo-CODI | 稠密网络（Phase 1） |
| 数据规模 | 800k | 50k（Phase 1） |
| 时间压力 | 高 | 中等 |
| 成功概率 | 中 | 高 |
