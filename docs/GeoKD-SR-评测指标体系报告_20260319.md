## 2. 评测指标体系架构

### 2.1 三类指标结构

评测指标体系分为三类，各有不同的用途和计算方式：

| 类别 | 类型 | 用途 | 指标 |
|-----|------|------|------|
| **确定性指标** | 规则计算 | 论文排名、显著性检验 | Overall Accuracy, Format Valid Rate, BLEU-4, ROUGE-L, Spatial F1 |
| **语义指标** | 嵌入相似度 | 辅助分析 | BERTScore (Precision/Recall/F1) |
| **大模型评估** | LLM打分 | 独立视角 | Reasoning/Completeness/Consistency/Overall (1-5分) |

### 2.2 指标用途说明

#### 确定性指标（用于排名）
- **Overall Accuracy**：核心评测指标，基于关键词匹配的核心答案正确率
- **Format Valid Rate**：模型输出格式有效性
- **BLEU-4**：4-gram文本相似度
- **ROUGE-L**：最长公共子序列相似度
- **Spatial F1**：空间关键词F1分数

#### 语义指标（辅助分析）
- **BERTScore**：基于BERT嵌入的语义相似度，能够捕捉同义词和语义变体

#### 大模型评估（独立视角）
- **Reasoning Quality**：推理逻辑质量（1-5分）
- **Answer Completeness**：答案完整性（1-5分）
- **Spatial Consistency**：空间描述一致性（1-5分）
- **Overall Score**：综合评分（1-5分）

## 3. 确定性指标详解

### 3.1 Overall Accuracy（整体准确率）

#### 定义
根据空间关系类型，判断模型输出的核心答案是否正确的样本比例。

#### 输入数据来源

| 数据项 | 来源 | 说明 |
|-------|------|------|
| prediction | 模型推理输出 | 待评测的文本 |
| reference | 数据集 `answer` 字段 | 标准答案 |
| spatial_type | 数据集 `spatial_relation_type` 字段 | 空间关系类型 |

#### 匹配规则详解

##### 3.1.1 方向关系匹配（Directional）

**匹配逻辑**：
1. 从参考答案和预测答案中提取方向关键词
2. 使用模糊匹配（支持方向别名）
3. 比较归一化后的方向是否一致

**方向关键词定义**：
```python
DIRECTION_KEYWORDS = [
    "东", "南", "西", "北", "东北", "东南", "西北", "西南",
    "东偏北", "东偏南", "西偏北", "西偏南",
    "北偏东", "北偏西", "南偏东", "南偏西",
    "正北", "正南", "正东", "正西"
]
```

**方向别名映射**（模糊匹配）：
```python
DIRECTION_ALIASES = {
    "东": ["东", "正东"],
    "南": ["南", "正南"],
    "西": ["西", "正西"],
    "北": ["北", "正北"],
    "东北": ["东北", "东偏北", "北偏东"],
    "东南": ["东南", "东偏南", "南偏东"],
    "西北": ["西北", "西偏北", "北偏西"],
    "西南": ["西南", "西偏南", "南偏西"]
}
```

**匹配示例**：
| 参考答案 | 预测答案 | 结果 | 说明 |
|---------|---------|------|------|
| "北京市在天津市的东南方向" | "东南" | 正确 | 方向一致 |
| "北京市在天津市的东南方向" | "东偏南" | 正确 | 别名匹配 |
| "北京市在天津市的东南方向" | "南方" | 错误 | 方向不一致 |

##### 3.1.2 拓扑关系匹配（Topological）

**匹配逻辑**：
1. 从参考答案和预测答案中提取拓扑关系类型
2. 比较拓扑关系类型是否一致（区分5种类型）
3. 只有类型一致才判定为正确

**拓扑关系类型映射**：
```python
TOPOLOGY_TYPE_MAP = {
    "within": [
        "在...内部", "位于...内", "在...里面", "属于", "包含于", "境内",
        "位于", "内部", "被包含"
    ],
    "contains": [
        "包含", "内部有", "有...在", "涵盖", "境内有"
    ],
    "adjacent": [
        "相邻", "毗邻", "接壤", "邻近", "交界", "边界"
    ],
    "disjoint": [
        "不相交", "分离", "相离", "没有重叠", "不存在"
    ],
    "overlap": [
        "重叠", "部分重叠", "交叉"
    ]
}
```

**匹配规则说明**：
- 按关键词长度降序排序，优先匹配长关键词避免误匹配
- 例如"在...内部"优先于"内部"匹配

**匹配示例**：
| 参考答案 | 预测答案 | 参考类型 | 预测类型 | 结果 |
|---------|---------|---------|---------|------|
| "江苏省与浙江省相邻" | "它们是相邻的" | adjacent | adjacent | 正确 |
| "江苏省与浙江省相邻" | "江苏省位于浙江省内部" | adjacent | within | 错误 |
| "北京市位于河北省内部" | "北京在河北境内" | within | within | 正确 |

##### 3.1.3 距离关系匹配（Metric）

**匹配逻辑**：
1. 从文本中提取距离数值
2. 计算预测值与参考值的偏差
3. 判断是否在容差范围内

**容差公式**：
```
tolerance = max(ref_dist × 0.15, 50)  # ±15% 或 ±50km（取较大值）
```

**容差设计原理**：
- 对于短距离（如100公里），15% = ±15公里，使用±50km兜底
- 对于长距离（如2500公里），15% = ±375公里，使用百分比容忍

**匹配示例**：
| 参考答案 | 预测答案 | 容差范围 | 结果 |
|---------|---------|---------|------|
| "距离约1200公里" | "1150公里" | 1020-1380km | 正确 |
| "距离约1200公里" | "1300公里" | 1020-1380km | 正确 |
| "距离约1200公里" | "1400公里" | 1020-1380km | 错误 |
| "距离约100公里" | "80公里" | 50-150km | 正确 |

##### 3.1.4 复合关系匹配（Composite）

**匹配逻辑**：
- 使用AND逻辑：方向和距离都必须正确
- `correct = direction_match AND distance_match`

**匹配示例**：
| 参考答案 | 预测答案 | 方向 | 距离 | 结果 |
|---------|---------|------|------|------|
| "上海市位于北京市东南方向约1200公里" | "东南方向，1150公里" | 正确 | 正确 | 正确 |
| "上海市位于北京市东南方向约1200公里" | "南方，1200公里" | 错误 | 正确 | 错误 |
| "上海市位于北京市东南方向约1200公里" | "东南方向，2000公里" | 正确 | 错误 | 错误 |

### 3.2 Format Valid Rate（格式有效率）

#### 定义
模型输出能够被正确解析的样本比例。

#### 判定标准
```python
def is_valid_format(text: str) -> bool:
    # 1. 非空且长度>=2
    if not text or len(text.strip()) < 2:
        return False

    # 2. 不是错误信息
    if text.startswith("ERROR:"):
        return False

    # 3. 包含有意义的中文或数字
    has_chinese = bool(re.search(r'[\u4e00-\u9fff]', text))
    has_number = bool(re.search(r'\d', text))

    return has_chinese or has_number
```

### 3.3 BLEU-4（文本相似度）

#### 定义
基于n-gram精确率几何平均的文本相似度指标，用于衡量生成文本与参考答案的词汇重叠程度。

#### 计算公式
```
BLEU = BP × exp(Σ(w_n × log(p_n)) / 4)

其中：
- BP (Brevity Penalty) = exp(1 - r/c) if c < r else 1
- p_n = n-gram精确率 (n=1,2,3,4)
- w_n = 1/4 (权重均等)
```

#### 实现特点
- 使用字符级分词（适用于中文）
- 支持平滑处理零精确率情况

---

### 3.4 ROUGE-L（最长公共子序列）

#### 定义
基于最长公共子序列(LCS)的文本相似度指标，同时考虑精确率和召回率。

#### 计算公式
```
F1 = 2 × P × R / (P + R)

其中：
- P (Precision) = LCS长度 / 预测文本长度
- R (Recall) = LCS长度 / 参考文本长度
```

#### 实现特点
- 使用动态规划算法计算LCS
- 能够捕捉词汇顺序信息

## 4. 语义指标

### 4.1 BERTScore（语义相似度）

#### 定义
基于预训练语言模型（BERT）嵌入的语义相似度评估指标。通过计算预测文本和参考答案在语义空间中的相似度，能够捕捉词汇变体、同义表达等语义层面的匹配。

#### 模型选择
- **推荐模型**：`bert-base-chinese`
- **原因**：专门针对中文优化，能够捕捉中文语义

#### 计算公式
```
Precision = (1/|pred|) × Σ max_sim(pred_i, ref)
Recall = (1/|ref|) × Σ max_sim(ref_j, pred)
F1 = 2 × P × R / (P + R)
```

#### 特点说明

| 特点 | 说明 |
|-----|------|
| **语义感知** | 能够识别同义词、近义词，如"东边"和"东方" |
| **上下文相关** | 基于BERT上下文嵌入，理解词语在句子中的含义 |
| **鲁棒性** | 对文本长度变化和表述差异具有较好的鲁棒性 |
| **中文支持** | 使用 bert-base-chinese 模型，专门针对中文优化 |

## 5. 大模型评估（独立视角）

### 5.1 评估维度

| 维度 | 评分范围 | 说明 |
|-----|---------|------|
| **Reasoning Quality** | 1-5分 | 推理逻辑质量 |
| **Answer Completeness** | 1-5分 | 答案完整性 |
| **Spatial Consistency** | 1-5分 | 空间描述一致性 |
| **Overall Score** | 1-5分 | 综合评分 |

### 5.2 评分标准

#### Reasoning Quality（推理质量）
- **5分**：推理链完整、逻辑清晰，步骤之间有明确的因果关系
- **4分**：推理基本正确，有小瑕疵但不影响结论
- **3分**：推理方向正确，但关键步骤缺失或跳跃
- **2分**：推理有明显错误，但有一定逻辑
- **1分**：推理完全错误或无推理过程

#### Answer Completeness（答案完整性）
- **5分**：完整回答问题，包含所有必要信息，表述清晰
- **4分**：基本完整，有轻微遗漏但不影响理解
- **3分**：部分回答，关键信息缺失，需要补充
- **2分**：回答不完整，缺少重要内容
- **1分**：未给出有效答案或答非所问

#### Spatial Consistency（空间一致性）
- **5分**：空间描述完全正确，方向/位置/距离准确
- **4分**：基本正确，有小偏差但不影响主要结论
- **3分**：部分正确，存在空间概念混淆
- **2分**：空间描述有明显错误
- **1分**：空间描述完全错误

### 5.3 重要说明

> **注意**：大模型评估结果仅作为独立参考视角，**不参与论文排名**。排名主要依据确定性指标。
