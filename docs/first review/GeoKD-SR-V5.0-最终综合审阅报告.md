## 六、关键发现深度分析

### 6.1 最核心问题：公式与代码不一致（S1）

**问题描述**：
- C1空间关系蒸馏损失的公式声明使用Forward KL
- 但代码实现实际使用的是Reverse KL
- 两者在知识蒸馏中的行为完全不同

**影响分析**：
- 学术严谨性问题
- 可能导致实验结果与预期不符
- 审稿人可能质疑方法的有效性

**修改建议**：
```python
# 修正方案1：修改公式声明
# 将公式改为：L_SRD = Σ w(r_i) × KL(P_S^i || P_T^i)

# 修正方案2：修改代码实现
def spatial_relation_distillation_loss(...):
    p_teacher = F.softmax(teacher_logits / T, dim=-1)
    p_student = F.log_softmax(student_logits / T, dim=-1)
    # Forward KL
    kl_loss = F.kl_div(p_student, p_teacher, reduction='none')
    ...
```

**问题描述**：

- 例如："北京在上海的西北方向" vs "上海在北京的东南方向"

### 6.3 实验设计核心缺陷：消融变体缺失（S4）

**问题描述**：
- C1使用预设权重[1.5, 1.3, 1.0, 1.8]
- 但没有对比实验验证权重的必要性
- Exp3 vs Exp2的提升可能来自"引入权重"而非"差异化权重"

**修改建议**：添加Exp3a消融变体

| 实验配置 | 方法 | 说明 |
|---------|------|------|
| Exp3 | B2 + C1 (Custom Weights) | 预设权重[1.5, 1.3, 1.0, 1.8] |
| **Exp3a** | B2 + C1 (Uniform Weights) | 均匀权重[1.0, 1.0, 1.0, 1.0] |

**验证逻辑**：
- Exp3 > Exp3a → 证明"差异化权重"有效
- Exp3a ≈ Exp3 → 权重设计无效，需要重新设计
- Exp3a > Exp2 → 证明"引入权重参数"本身有效

### 6.4 数据处理核心缺陷：循环依赖（S10）

**问题描述**：
- 方案使用Qwen2.5-7B（教师模型）生成8,000条训练数据
- 然后用这些数据训练学生模型
- 再用教师模型蒸馏学生模型
- 存在概念循环：学生模型被用来生成自己的训练数据

**风险场景**：
```
教师模型错误示例：
- 问：北京在上海的什么方向？
- 教师生成数据可能包含：北京在上海的"正西方向"（错误）
- 学生模型学习这个错误模式
- 蒸馏过程中，教师模型继续强化这个错误
```

**修改建议**：
1. 增加独立数据来源：至少30%的数据来自非教师模型来源
2. 添加教师模型错误率评估
3. 在论文中明确声明数据生成方法的局限性
