> **审阅日期**: 2026年3月2日
> **审阅人**: 评测体系与指标审阅专家
> **审阅维度**: 评测体系与指标设计
> **已有问题基数**: 16个（来自V5.0审阅报告第十二章）

## 一、执行摘要

本次审阅针对GeoKD-SR实验设计方案的评测体系和指标设计，在已有16个问题基础上，**新增发现4个严重问题**和**6个中等问题**，并对已有问题8提供了重要补充。

**关键发现**：
1. **地理特异性指标严重缺失** - 通用指标(BLEU/ROUGE)无法评估空间推理核心能力
2. **GPT-4数据泄露风险极高** - 用Qwen生成数据、GPT-4评测存在严重泄露
3. **评测基准缺少关键对比** - 没有与空间推理SOTA模型对比
4. **主要评估指标选择不当** - "答案准确率"无法体现推理质量

## 二、新发现问题列表

### 2.1 严重级别问题（必须修改）

#### 问题17：通用指标对空间推理的区分度严重不足
- **位置**: 7.2.1节 性能指标（第921-928行）
- **问题描述**:
  - BLEU/ROUGE是针对翻译/摘要任务的通用指标
  - **致命缺陷**：无法区分语义相反但词汇高度重叠的答案
  - **举例**：
    ```
    标准答案："北京在上海的西北方向"
    模型答案："上海在北京的东南方向"
    BLEU分数：高（词汇重叠度高）
    实际评估：错误（方向完全相反）
    ```
  - 这类指标无法评估空间推理的核心能力
- **影响**:
  - 评测结果虚高，无法反映真实空间推理能力
  - 审稿人会强烈质疑评测体系的有效性
  - 无法证明方法的有效性
- **建议**: **必须添加地理特异性指标**
  ```
  新增指标（作为主要评估指标）：
  1. 方向错误率(DER)：统计八方向判断错误的百分比
  2. 拓扑混淆矩阵：针对包含/相邻/相交/包含于的混淆矩阵
  3. 距离误差MAPE：度量关系的平均绝对百分比误差
  4. 空间关系准确率(SRA)：按四种关系类型分别计算准确率

  主要评估指标改为：SRA（空间关系准确率）
  ```

#### 问题18：GPT-4评估存在严重数据泄露风险
- **位置**: 7.3节 LLM辅助评测（第956-994行）
- **问题描述**:
  - **高风险场景**：用Qwen2.5-7B生成训练数据 → 再用GPT-4评测
  - GPT-4在预训练时可能见过类似的地理问题和答案
  - 训练数据和评测数据都是中国地理，分布高度相似
  - **已有问题9的补充和升级**：问题9提到了数据泄露风险，但未强调其严重性
- **影响**:
  - 评测结果严重虚高，无法真实反映模型泛化能力
  - **投稿会被直接拒稿**：这是严重的学术诚信问题
  - 无法证明方法的实际有效性
- **建议**:
  ```
  P0级修复措施：
  1. 立即添加n-gram重叠检测，量化数据泄露程度
  2. 引入开源Judge模型（JudgeLM-7B、Prometheus）作为补充
  3. 人工验证100-200题作为金标准（必须）
  4. 在论文中明确声明评测集构建时间和来源
  5. 构造分布外测试样本（非中国地理）
  ```

#### 问题19：评测采样策略存在严重统计缺陷
- **位置**: 7.3.2节 评估采样策略（第987-994行）
- **问题描述**:
  - **采样量不足**：130题仅占1000题的13%，置信区间过宽（约±8%）
  - **采样不均衡**：组合推理（最难）采样比例反而更低（15% vs 10%）
  - **分布不一致**：采样比例与测试集原始分布（D1/D2/D3 = 40/40/20）不匹配
  - **分层不科学**：只按关系类型分层，未按难度分层
- **影响**:
  - 评估结果不可靠，无法代表整体性能
  - 难题的评估结果统计功效不足
  - 统计检验的可信度低
- **建议**:
  ```
  修改后的采样策略：
  1. 总采样量：300题（30%）以获得95%置信区间±5%
  2. 三维分层：按D1/D2/D3（各120/120/60题）× 关系类型 × 难度
  3. 难度平衡：Easy/Medium/Hard各占30/50/20题
  4. 报告95%置信区间和标准误
  5. 使用分层抽样确保代表性
  ```

#### 问题20：缺少基准模型和SOTA对比
- **位置**: 第七章整节缺失
- **问题描述**:
  - 只有消融实验对比（Exp1-Exp9），没有与领域内SOTA对比
  - **缺少空间推理专用模型对比**：
    - GeoLLM（地理大语言模型）
    - Spatial-BERT（空间预训练模型）
    - Location-aware BERT
  - **缺少通用LLM对比**：
    - GPT-3.5、Llama-2-7B、Qwen-7B（无蒸馏）
  - **缺少蒸馏方法对比**：
    - MiniLLM、DistilBERT等通用蒸馏方法
- **影响**:
  - 无法证明方法的先进性
  - 无法证明空间关系感知蒸馏的额外价值
  - **投稿会被拒稿**：缺少基线对比是方法类论文的致命缺陷
- **建议**:
  ```
  必须添加的对比实验：
  1. 空间推理SOTA：GeoLLM、Spatial-BERT
  2. 通用LLM：GPT-3.5、Llama-2-7B、Qwen-7B
  3. 蒸馏方法：MiniLLM、DistilBERT
  4. 报告与SOTA的性能差距
  5. 分析GeoKD-SR的独特优势
  ```

### 2.2 中等级别问题（建议修改）

#### 问题21：主要评估指标"答案准确率"过于粗糙
- **位置**: 7.2.1节（第923行）
- **问题描述**:
  - "答案准确率(Acc)"作为主要评估指标过于简单粗暴
  - **无法区分**：
    - 推理步骤正确但最终答案错误（应得部分分数）
    - 答案正确但推理过程错误（不应得满分）
  - 与deepseek审阅中提到的问题类似但更具体
- **影响**:
  - 低估模型能力（特别是推理能力）
  - 无法体现推理链蒸馏的价值
  - 评估结果不够细致
- **建议**:
  ```
  改为主要指标：推理准确率(RA)
  RA = (推理步骤全对×1.0 + 推理步骤部分对×0.5 + 最终答案对×0.3) / 总题数

  辅助指标：答案准确率(AA) - 仅作为参考
  ```

#### 问题22：GPT-4评估Prompt设计存在多处缺陷
- **位置**: 7.3.1节 GPT-4评估设置（第960-983行）
- **问题描述**:
  - **缺少参考推理链**：Prompt中没有提供标准推理链，GPT-4无法评估推理质量
  - **评分维度权重不明**：4个维度（准确性/完整性/推理质量/语言流畅性）的权重不清楚
  - **缺少Few-shot示例**：没有示例来引导GPT-4的评分标准
  - **缺少CoT要求**：没有强制GPT-4给出推理过程
  - **对已有问题8的补充**：问题8提到了GPT-4的可靠性问题，但未指出Prompt设计的具体缺陷
- **影响**:
  - GPT-4评估不够准确和稳定
  - 评估结果的可信度降低
  - 无法评估推理链的质量
- **建议**:
  ```python
  IMPROVED_EVALUATION_PROMPT = """
  你是一个地理空间推理专家。请评估以下答案的质量。

  问题: {question}
  标准答案: {reference}
  标准推理链: {reference_reasoning}  # 新增
  模型答案: {prediction}
  模型推理链: {prediction_reasoning}  # 新增

  评估标准：
  1. 推理准确性(40%)：推理步骤是否正确
  2. 最终答案(30%)：答案是否正确
  3. 完整性(20%)：是否遗漏关键信息
  4. 表达清晰度(10%)：是否容易理解

  Few-shot示例：
  [示例1] 满分答案（推理5、答案5、完整性5、清晰度5）
  [示例2] 部分正确答案（推理3、答案3、完整性2、清晰度4）

  评估流程：
  1. 逐步对比推理链
  2. 验证最终答案
  3. 检查完整性
  4. 评估表达清晰度

  请按上述标准评分（1-5分），并给出详细理由。
  """
  ```

#### 问题23：错误类型分类缺少空间推理特有错误
- **位置**: 7.4.1节 错误类型分类（第1000-1006行）
- **问题描述**:
  - 只有5种通用错误类型，缺少空间推理特有的错误
  - **缺少的关键错误类型**：
    - 方向混淆：八方向之间的误判（特别是相邻方向如东北vs西北）
    - 拓扑关系误判：包含/相邻/相交/包含于的混淆
    - 度量误差等级：误差<10% / 10-50% / >50%的分级
    - 实体位置错误：地理实体坐标错误
    - 多步推理断裂：推理链中间某步出错
- **影响**:
  - 错误分析不够深入
  - 无法针对性改进模型
  - 错误模式不够清晰
- **建议**:
  ```
  扩展错误类型为10种：
  空间推理特有错误（新增）：
  1. 方向混淆：八方向误判（分相邻/相对方向）
  2. 拓扑关系误判：包含/相邻/相交/包含于混淆
  3. 度量误差严重：距离误差>50%
  4. 实体位置错误：地理实体坐标错误
  5. 多步推理断裂：推理链中间某步出错
  6. 坐标计算错误：经纬度计算错误

  通用错误（保留）：
  7. 空间关系错误
  8. 实体识别错误
  9. 幻觉错误
  10. 格式错误
  ```

#### 问题24：评测基准GeoSR-Bench代表性不足
- **位置**: 7.1节 评测基准（第909-915行）
- **问题描述**:
  - **规模偏小**：1000题不足以覆盖空间推理的多样性
  - **维度划分不明确**：D1（空间关系理解）/D2（空间推理能力）/D3（地理知识融合）的划分标准不清晰
  - **缺少多样性分析**：

  ```
  基准增强：
  1. 扩展到3000题（每种关系类型>500题）
  2. 明确D1/D2/D3划分标准并提供示例
  3. 进行难度标注和分布分析
  4. 确保地理实体分布均匀（不只东部）
  ```

## 三、对已有问题的补充

### 3.1 对问题8（GPT-4辅助评测可靠性）的补充

**原问题8**：GPT-4评测存在位置偏差、冗长偏差、自我增强偏差等已知问题

**补充内容**：
1. **人机一致性验证必须具体化**：
   - 请2-3位地理专家评估100题
   - 计算Cohen's Kappa系数（目标>0.7）
   - 如果Kappa<0.7，GPT-4评估不可用

2. **GPT-4稳定性测试必须量化**：
   - 同一问题评估3次
   - 计算标准差（目标<0.5分）
   - 报告稳定性指标

3. **建议使用评估指标集**：
   - Pearson相关系数（人机相关性）
   - Spearman相关系数（排名相关性）
   - Cohen's Kappa（一致性）
   - MAE/RMSE（误差）

### 3.2 对问题9（数据泄露风险）的严重性强调

**原问题9**：使用Qwen2.5-7B生成训练数据，GPT-4评测，但未评估数据泄露风险

**补充内容**（升级为严重问题）：
1. **这是学术诚信问题**，可能导致直接拒稿
2. **必须添加的具体措施**：
   - n-gram重叠检测（4-gram、5-gram）
   - 计算重叠率（阈值<5%）
   - 如果重叠率>10%，必须更换评测方法

3. **替代方案**：
   - 使用开源Judge模型（JudgeLM-7B、Prometheus-7B）
   - 人工评估（200题，2-3位专家）
   - 构造分布外测试集

## 六、总结

GeoKD-SR的评测体系在框架上是完整的，但在**地理特异性**和**评估可靠性**方面存在严重不足：

1. **通用指标(BLEU/ROUGE)不适用于空间推理**，必须添加方向错误率、拓扑混淆矩阵等地理特异性指标
2. **GPT-4数据泄露风险极高**，需要引入开源Judge模型和人工验证
