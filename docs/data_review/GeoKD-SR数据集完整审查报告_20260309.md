### 4.2 泄露示例

**示例1: relation_type泄露**
```json
{
  "step": 2,
  "name": "spatial_relation_extraction",
  "action": "classify_relation",
  "content": "识别问题中的空间关系关键词'位于...境内'，判定为拓扑关系中的包含关系查询。",
  "relation_type": "topological"  // ← 直接泄露任务类型！
}
```

**示例2: calculation_result泄露**
```json
{
  "step": 4,
  "name": "spatial_calculation",
  "action": "determine_topology",
  "content": "根据行政区划数据，长沙市是湖南省的省会...",
  "calculation_result": "长沙不在陕西省内"  // ← 直接泄露答案线索！
}
```

### 4.3 泄露影响分析

| 影响类型 | 说明 |
|----------|------|
| 捷径学习 | 模型可直接从`relation_type`推断任务类型，无需理解问题 |
| 答案泄露 | `calculation_result`直接给出推理结论，跳过推理过程 |
| 评估失效 | 无法真实评估模型的空间推理能力 |
| 泛化能力下降 | 模型学习泄露模式而非真正的空间推理 |
