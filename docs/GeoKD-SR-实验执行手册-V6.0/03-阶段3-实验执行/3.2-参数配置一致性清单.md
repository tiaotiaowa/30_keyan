### 3.2 训练中验证

```python
# 在训练脚本中添加配置验证
def validate_experiment_config(exp_config):
    """验证实验配置的完整性"""
    required_fields = [
        'experiment_name',
        'model_name',
        'train_data',
        'test_data',
        'loss_fn',
        'learning_rate',
        'batch_size',
        'num_epochs',
        'seed',
        'output_dir'
    ]

    missing = [f for f in required_fields if f not in exp_config]
    if missing:
        raise ValueError(f"缺少必要配置项: {missing}")

    # 验证统一参数
    assert exp_config['learning_rate'] == 2e-5, "学习率必须为2e-5"
    assert exp_config['batch_size'] == 16, "批大小必须为16"
    assert exp_config['num_epochs'] == 3, "训练轮数必须为3"

    # 验证LoRA配置
    if 'lora_r' in exp_config:
        assert exp_config['lora_r'] == 16, "LoRA rank必须为16"
    if 'lora_alpha' in exp_config:
        assert exp_config['lora_alpha'] == 32, "LoRA alpha必须为32"

    return True
```

### 3.3 执行后验证

```bash
#!/bin/bash
# post_experiment_check.sh - 实验后检查脚本

EXPERIMENT_DIR=$1

echo "=== 实验后检查: $EXPERIMENT_DIR ==="

# 1. 检查模型文件

   [ -f "$EXPERIMENT_DIR/best_model/adapter_model.bin" ]; then
    echo "✓ 模型文件存在"
else
    echo "✗ 模型文件缺失"
    exit 1
fi

# 2. 检查日志文件
echo "[2] 检查日志文件..."
if [ -f "$EXPERIMENT_DIR/trainer_log.jsonl" ]; then
    echo "✓ 训练日志存在"
    # 检查是否有NaN损失
    if grep -q "nan\|NaN\|inf" "$EXPERIMENT_DIR/trainer_log.jsonl"; then
        echo "⚠ 检测到NaN/Inf损失值"
    fi
else
    echo "✗ 训练日志缺失"
    exit 1
fi

# 3. 检查配置文件
echo "[3] 检查配置文件..."
if [ -f "$EXPERIMENT_DIR/config.json" ]; then
    echo "✓ 配置文件存在"
else
    echo "✗ 配置文件缺失"
    exit 1
fi

# 4. 检查评估结果
echo "[4] 检查评估结果..."
if [ -f "$EXPERIMENT_DIR/eval_results.json" ]; then
    echo "✓ 评估结果存在"
    python -c "
import json

    results = json.load(f)
    print(f\"  准确率: {results.get('accuracy', 'N/A')}\")

else
    echo "⚠ 评估结果缺失，需要运行评估脚本"
fi

echo "=== 检查完成 ==="
```

## 五、常见配置错误

### 5.1 错误1：LoRA rank不一致

**错误现象**:
```
ValueError: LoRA rank mismatch: expected 16, got 8
```

**解决方案**: 确保所有实验使用 `r=16`

---

### 5.2 错误2：温度参数不一致

**错误现象**:
```
Exp2和Exp3使用不同温度，导致结果不可比
```

**解决方案**: 所有使用KD的实验统一使用 `T=2.0`（Exp8除外，因为其使用温度调度）

---

### 5.3 错误3：数据集不一致

**错误现象**:
```
Exp1使用train_v1.json，Exp2使用train_v2.json
```

**解决方案**: 使用符号链接确保所有实验指向同一数据文件

```bash
cd D:/30_keyan/GeoKD-SR/data
ln -s GeoSR-Train-v1.0.json GeoSR-Train.json
```
