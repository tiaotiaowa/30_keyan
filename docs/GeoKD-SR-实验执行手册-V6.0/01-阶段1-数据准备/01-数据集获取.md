## 二、GeoSR-Bench测试集获取

### 2.1 数据集说明

GeoSR-Bench是一个专门用于评估地理空间推理能力的基准测试集，包含3000道经过精心设计的测试题目。

### 2.2 获取方式

#### 方式1：从项目仓库获取

```bash
# 假设数据集存储在项目仓库中
cd D:/30_keyan/GeoKD-SR
cp data/GeoSR-Bench-3000.jsonl ./data/test/
```

#### 方式2：从外部数据源下载

```bash
# 如果数据存储在外部服务器
wget https://example.com/datasets/GeoSR-Bench-3000.jsonl -O data/test/GeoSR-Bench-3000.jsonl
```

#### 方式3：从共享网盘获取

- 检查项目文档中的网盘链接
- 下载并解压到指定目录

## 四、训练数据集获取

### 4.1 获取方式

使用GLM-5 API自动生成:

```bash
# 生成训练数据
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --dev_count 800 \
    --test_count 3000 \
    --output data/geosr_chain/

# 需要配置智谱API密钥
export ZHIPUAI_API_KEY=your_api_key_here
```

## 五、数据格式规范 (V2.0)

### 5.1 标准格式

```json
{
  "id": "geosr_001",
  "spatial_relation_type": "directional",
  "topology_subtype": "within",
  "question": "北京在上海的什么方向？",
  "answer": "西北方向",
  "reasoning_chain": [
    {
      "step": 1,
      "type": "entity_recognition",
      "content": "识别实体：北京、上海"
    },
    {
      "step": 2,
      "type": "coordinate_lookup",
      "content": "北京坐标：(39.90, 116.40), 上海坐标：(31.23, 121.47)"
    },
    {
      "step": 3,
      "type": "direction_calculation",
      "content": "计算相对方向：北京位于上海西北方向"
    }
  ],
  "entities": [
    {
      "name": "北京",
      "type": "city",
      "coordinates": [116.40, 39.90]
    },
    {
      "name": "上海",
      "type": "city",
      "coordinates": [121.47, 31.23]
    }
  ],
  "spatial_tokens": ["北京", "上海", "西北", "方向"],
  "entity_to_token": {
    "北京": {
      "char_start": 0,
      "char_end": 2,
      "type": "city"
    },
    "上海": {
      "char_start": 3,
      "char_end": 5,
      "type": "city"
    }
  },
  "difficulty": "easy",
  "difficulty_score": 1.5
}
```

### 5.2 多选题格式

```json
{
  "question": "题目内容",
  "options": ["A. xxx", "B. xxx", "C. xxx", "D. xxx"],
  "answer": "A",
  "reasoning": "选择A的原因..."
}
```

### 5.3 开放题格式

```json
{
  "question": "题目内容",
  "answer": "参考答案",
  "reasoning": "解题思路..."
}
```

### 5.4 V2.0新增字段说明

| 字段名 | 类型 | 说明 | 示例 |
|--------|------|------|------|
| topology_subtype | string | 拓扑子类型 | within/contains/adjacent/disjoint/overlap |
| difficulty_score | float | 难度分数 | 1.0-5.0范围 |
| spatial_tokens | array | 空间关键词 | ["北京", "上海", "西北"] |
| entity_to_token | object | 实体Token映射 | {"北京": {"char_start": 0, ...}} |

## 七、常见问题排查

### 问题1：下载失败

**现象**：网络错误或链接失效

**解决方案**：
1. 检查网络连接
2. 尝试备用下载链接
3. 联系数据提供者

### 问题2：文件损坏

**现象**：文件无法打开或解析失败

**解决方案**：
1. 重新下载数据
2. 检查下载完整性（MD5/SHA256）
3. 使用备份文件

### 问题3：编码问题

**现象**：中文显示乱码

**解决方案**：
1. 确认文件编码为UTF-8
2. 转换编码格式
3. 使用正确的解码参数

### 问题4：格式不匹配

**现象**：字段名称或结构不一致

**解决方案**：
1. 使用格式转换脚本
2. 参考标准格式进行字段映射
3. 记录格式差异并统一
