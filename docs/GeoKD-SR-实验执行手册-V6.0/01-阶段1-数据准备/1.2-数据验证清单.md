# 1.2 数据验证清单

## 概述

本文档提供GeoKD-SR数据集的完整验证规则和检查清单，确保所有生成的训练数据符合质量标准。

---

## 一、6层验证机制概述

GeoKD-SR数据集采用6层验证机制确保数据质量：

| Level | 验证内容 | 通过标准 | 说明 |
|-------|---------|---------|------|
| **L1** | 格式验证 | 100% | JSON有效性、必需字段存在性 |
| **L2** | 语义验证 | 100% | 字段类型正确、枚举值有效性 |
| **L3** | 空间关系验证 | >=95% | 关键词检测匹配relation_type |
| **L4** | 坐标验证 | 100% | 经度73.66-135.05度E，纬度3.86-53.55度N |
| **L5** | 推理链验证 | >=90% | 5步结构完整性、逻辑一致性 |
| **L6** | 去重验证 | 100% | 余弦相似度 < 0.9 |

## 二、推理链逻辑一致性验证（D3-1）

### 2.1 验证规则

#### 规则1：步骤完整性验证

每条数据必须包含完整的5步推理链，不能跳过任何步骤。

**验证代码示例：**
```python
def validate_reasoning_chain_completeness(data):
    """验证推理链完整性"""
    required_steps = [1, 2, 3, 4, 5]
    required_names = [
        "entity_identification",
        "spatial_relation_extraction",
        "coordinate_retrieval",
        "spatial_calculation",
        "answer_generation"
    ]

    reasoning_chain = data.get("reasoning_chain", [])

    # 检查步骤数量
    if len(reasoning_chain) != 5:
        return False, f"推理链步骤数量错误: 期望5步, 实际{len(reasoning_chain)}步"

    # 检查步骤编号
    actual_steps = [step["step"] for step in reasoning_chain]
    if actual_steps != required_steps:
        return False, f"步骤编号错误: {actual_steps}"

    # 检查步骤名称
    actual_names = [step["name"] for step in reasoning_chain]
    if actual_names != required_names:
        return False, f"步骤名称错误: {actual_names}"

    return True, "推理链完整性验证通过"
```

#### 规则2：步骤间逻辑连贯性验证

每个步骤的输出应作为下一步骤的输入。

**验证要点：**
- Step1识别的entities必须与Step3的coordinates键一致
- Step2的relation_type必须与Step4的计算方法匹配
- Step4的计算结果必须与Step5的answer数值一致

**验证代码示例：**
```python
def validate_reasoning_chain_coherence(data):
    """验证推理链逻辑连贯性"""
    chain = data["reasoning_chain"]

    # 提取各步骤信息
    step1_entities = set(chain[0].get("entities", []))
    step3_coords = set(chain[2].get("coordinates", {}).keys())
    step2_relation = chain[1].get("relation_type")
    step4_method = chain[3].get("method")
    step4_result = chain[3].get("result")
    step5_answer = chain[4].get("answer")

    # 验证: Step1 entities == Step3 coordinates keys
    if step1_entities != step3_coords:
        return False, f"实体不一致: Step1={step1_entities}, Step3={step3_coords}"

    # 验证: relation_type与method匹配
    relation_method_map = {
        "距离": ["haversine", "euclidean", "manhattan"],
        "方位": ["bearing"],
        "拓扑": ["adjacency"],
        "包含": ["containment"]
    }

    valid_methods = relation_method_map.get(step2_relation, [])
    if step4_method not in valid_methods:
        return False, f"方法不匹配: relation={step2_relation}, method={step4_method}"

    # 验证: Step4结果与Step5答案数值一致
    answer_numbers = extract_numbers_from_text(step5_answer)
    if answer_numbers and abs(answer_numbers[0] - step4_result) > step4_result * 0.05:
        return False, f"数值不一致: Step4={step4_result}, Step5={answer_numbers[0]}"

    return True, "逻辑连贯性验证通过"
```

#### 规则3：action字段验证

每个步骤必须使用正确的action值。

| Step | name | 正确action |
|------|------|-----------|
| 1 | entity_identification | extract_entities |
| 2 | spatial_relation_extraction | classify_relation |
| 3 | coordinate_retrieval | infer_entity_to_token |
| 4 | spatial_calculation | calculate |
| 5 | answer_generation | generate_answer |

## 三、实体一致性验证（D3-2）

### 3.1 entities字段验证

#### 验证规则1：entities必须包含question中的所有地理实体

**验证代码：**
```python
def validate_entities_completeness(data):
    """验证entities字段完整性"""
    question = data["question"]
    entities_field = data.get("entities", [])

    # 使用命名实体识别提取question中的地理实体
    extracted_entities = extract_geo_entities(question)

    # 检查是否所有实体都被包含
    missing_entities = set(extracted_entities) - set(entities_field)

    if missing_entities:
        return False, f"缺失实体: {missing_entities}"

    return True, "实体完整性验证通过"
```

#### 验证规则2：entities与entity_to_token键一致

**验证代码：**
```python
def validate_entity_token_mapping(data):
    """验证实体到Token映射一致性"""
    entities = data.get("entities", [])
    entity_to_token = data.get("entity_to_token", {})

    # 检查所有entities都有对应的token映射
    missing_mappings = set(entities) - set(entity_to_token.keys())

    if missing_mappings:
        return False, f"缺少Token映射: {missing_mappings}"

    # 检查token索引有效性
    for entity, mapping in entity_to_token.items():
        tokens = mapping.get("tokens", [])
        if not tokens or any(t < 0 for t in tokens):
            return False, f"无效的Token索引: {entity} -> {tokens}"

    return True, "实体Token映射验证通过"
```

### 3.2 coordinates验证

```python
def validate_difficulty_score(data):
    """验证难度分数范围"""
    score = data.get("difficulty_score")
    if score is None:
        return False, "缺少difficulty_score字段"
    if not isinstance(score, (int, float)):
```

### 4.4 entity_to_token映射完整性验证

```python
def validate_entity_to_token_completeness(data):
    """验证实体Token映射完整性"""
    entities = data.get("entities", [])
    entity_to_token = data.get("entity_to_token", {})

    entity_names = {e.get("name") for e in entities}
    mapped_names = set(entity_to_token.keys())

    missing = entity_names - mapped_names
    if missing:
        return False, f"缺少Token映射的实体: {missing}"

    for entity, mapping in entity_to_token.items():
        if "char_start" not in mapping or "token_indices" not in mapping:
            return False, f"映射不完整: {entity}"

    return True, "entity_to_token映射完整性验证通过"
```

```python
EXPERIMENT_REQUIREMENTS = {

             "entities", "spatial_tokens", "entity_to_token", "difficulty",
             "difficulty_score"]
}

def validate_experiment_compatibility(data):
    """验证实验兼容性"""
    results = {}
    for exp, required_fields in EXPERIMENT_REQUIREMENTS.items():
        missing = [f for f in required_fields if f not in data]
        results[exp] = {
            "compatible": len(missing) == 0,
            "missing_fields": missing
        }
    return results
```

```python
def validate_coordinate_ranges(data):
    """验证坐标范围"""
    coordinates = data.get("coordinates", {})

    for entity, coord in coordinates.items():
        lat = coord.get("lat")
        lon = coord.get("lon")

        if lat is None or lon is None:
            return False, f"{entity}: 缺少经纬度"

        if not (COORDINATE_BOUNDS["lat_min"] <= lat <= COORDINATE_BOUNDS["lat_max"]):
            return False, f"{entity}: 纬度超出范围 {lat} [{COORDINATE_BOUNDS['lat_min']}, {COORDINATE_BOUNDS['lat_max']}]"

        if not (COORDINATE_BOUNDS["lon_min"] <= lon <= COORDINATE_BOUNDS["lon_max"]):
            return False, f"{entity}: 经度超出范围 {lon} [{COORDINATE_BOUNDS['lon_min']}, {COORDINATE_BOUNDS['lon_max']}]"

    return True, "坐标范围验证通过"
```

### 8.2 批量数据验证

```python
def batch_validate_dataset(jsonl_file):
    """批量验证数据集"""
    results = {
        "total": 0,
        "passed": 0,
        "failed": 0,
        "errors": []
    }

    validators = [
        ("基础字段", validate_basic_fields),
        ("推理链完整性", validate_reasoning_chain_completeness),
        ("逻辑连贯性", validate_reasoning_chain_coherence),
        ("实体完整性", validate_entities_completeness),
        ("实体Token映射", validate_entity_token_mapping),
        ("坐标完整性", validate_coordinates_completeness),
        ("坐标范围", validate_coordinate_ranges),
        # V2.0新增验证器
        ("topology_subtype", validate_topology_subtype),
        ("difficulty_score", validate_difficulty_score),
        ("推理链结构", validate_reasoning_chain_structure),
        ("entity_to_token完整性", validate_entity_to_token_completeness),
        ("实验兼容性", validate_experiment_compatibility)
    ]

    with open(jsonl_file, 'r', encoding='utf-8') as f:
        for line_num, line in enumerate(f, 1):
            results["total"] += 1

            try:
                data = json.loads(line.strip())
                data_passed = True

                for name, validator in validators:
                    passed, msg = validator(data)
                    if not passed:
                        data_passed = False
                        results["errors"].append({
                            "line": line_num,
                            "validator": name,
                            "error": msg
                        })

                if data_passed:
                    results["passed"] += 1
                else:
                    results["failed"] += 1

            except Exception as e:
                results["failed"] += 1
                results["errors"].append({
                    "line": line_num,
                    "validator": "JSON解析",
                    "error": str(e)
                })

    return results
```

## 九、常见问题与解决方案

### 9.1 推理链问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 步骤缺失 | 生成模板不完整 | 检查prompt模板 |
| 步骤顺序错误 | 逻辑设计问题 | 严格按照5步顺序 |
| action字段错误 | 命名不规范 | 使用标准action值 |
| 逻辑不连贯 | 生成LLM理解偏差 | 增强prompt约束 |

### 9.2 实体标注问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 实体识别不全 | NER模型漏检 | 人工补充检查 |
| Token映射错误 | tokenizer不一致 | 使用统一tokenizer |
| 坐标缺失 | 数据源问题 | 补充坐标数据 |

### 9.3 坐标问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 超出中国范围 | 错误坐标源 | 使用权威数据源 |
| 纬度经度反了 | 输入错误 | 交换lat/lon |
| 精度不够 | 取整过早 | 保留4位小数 |

### 9.4 V2.0新增问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| topology_subtype缺失 | topological类型未标注 | 补充拓扑子类型字段 |
| difficulty_score超范围 | 评分标准不一致 | 使用1.0-5.0标准评分 |
| entity_to_token不完整 | 映射生成遗漏 | 检查char_start和token_indices |
| 实验兼容性不足 | 字段缺失 | 按实验需求补全字段 |
