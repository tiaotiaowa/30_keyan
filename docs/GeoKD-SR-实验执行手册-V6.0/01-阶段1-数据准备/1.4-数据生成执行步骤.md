## 三、Step 2: 生成数据

### 3.1 完整生成命令

```bash
# 生成完整数据集
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --dev_count 800 \
    --test_count 3000 \
    --output data/geosr_chain/ \
    --post_process
```

### 3.2 预期执行时间 (V7.0更新)

| 数据集 | 数量 | 预计时间 | 说明 |
|--------|------|----------|------|
| 训练集 | 8,000 | **2-4小时** | GLM-5 API调用 |
| 验证集 | 800 | **15-30分钟** | GLM-5 API调用 |
| 测试集 | 3,000 | **45-90分钟** | GLM-5 API调用 |
| 后处理 | - | **15-20分钟** | Token映射、验证 |
| **总计** | **11,800** | **3-6小时** | - |

### 3.3 分步生成（推荐）

如果担心中断，可以分步生成：

```bash
# 步骤1: 仅生成训练集
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --dev_count 0 \
    --test_count 0 \
    --output data/geosr_chain/

# 步骤2: 仅生成验证集
python scripts/generate_data_glm5.py \
    --train_count 0 \
    --dev_count 800 \
    --test_count 0 \
    --output data/geosr_chain/

# 步骤3: 仅生成测试集
python scripts/generate_data_glm5.py \
    --test_only \
    --test_count 3000 \
    --output data/geosr_chain/
```

### 3.4 监控进度

```bash
# 查看当前生成进度
python -c "
import json
with open('data/geosr_chain/train.jsonl', 'r', encoding='utf-8') as f:
    count = sum(1 for _ in f)
print(f'已生成训练数据: {count} 条')
"
```

## 六、故障排查

### 6.1 API调用失败

**错误信息:**
```
[错误] API调用失败: 429 - Too Many Requests
```

**解决方案:**
```bash
# 方案1: 增加请求间隔
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --api_delay 2  # 每次请求间隔2秒

# 方案2: 分批生成
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --batch_size 100  # 每批100条，中间暂停
```

### 6.2 内存不足

**错误信息:**
```
[错误] MemoryError: Unable to allocate array
```

**解决方案:**
```bash
# 分批生成，避免内存溢出
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --batch_size 500 \
    --save_interval 500  # 每500条保存一次
```

### 6.3 Token映射失败

**错误信息:**
```
[警告] entity_to_token映射失败: tokenizer未加载
```

**解决方案:**
```bash
# 确保transformers已安装
pip install transformers>=4.30.0

# 或跳过后处理
python scripts/generate_data_glm5.py \
    --train_count 8000 \
    --no_post_process

# 然后单独运行映射
python scripts/generate_entity_mapping.py \
    --input data/geosr_chain/train.jsonl \
    --output data/geosr_chain/train_mapped.jsonl
```

### 6.4 验证失败

**错误信息:**
```
[失败] L5推理链验证: 通过率 85% < 90%
```

**解决方案:**
```bash
# 1. 查看详细错误
python scripts/validate_data.py \
    --input data/geosr_chain/train.jsonl \
    --verbose \
    --text_report reports/errors.txt

# 2. 重新生成失败的数据
python scripts/generate_data_glm5.py \
    --retry_failed \
    --input reports/failed_records.json
```

### 6.5 topology_subtype验证失败

**错误信息:**
```
[失败] topology_subtype验证: 无效的子类型 unknown
```

**解决方案:**
```bash
# 检查有效的拓扑子类型
python -c "
valid_subtypes = ['within', 'contains', 'adjacent', 'disjoint', 'overlap']
print(f'有效的拓扑子类型: {valid_subtypes}')
"
```

### 6.6 difficulty_score超出范围

**错误信息:**
```
[失败] difficulty_score超出范围: 5.5
```

**解决方案:**
```bash
# 重新计算difficulty_score
python scripts/recalculate_difficulty.py \
    --input data/geosr_chain/train.jsonl \
    --output data/geosr_chain/train_fixed.jsonl
```
