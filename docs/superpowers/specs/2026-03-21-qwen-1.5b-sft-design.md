## 1. 概述

### 1.1 目标

为 GeoKD-SR 项目实现 **Exp1 Direct-SFT 基线实验**，使用 LoRA 对 Qwen2.5-1.5B-Instruct 进行监督微调。

**核心目标**：
1. 在 Windows 6GB 环境验证训练流程可行性
2. 迁移到阿里云 24GB 环境完成正式实验
3. 对比 splits(无坐标) 和 split_coords(有坐标) 两个数据集版本的效果

### 1.2 关键约束

| 约束 | 说明 |
|------|------|
| 显存限制 | Windows: 6GB / 阿里云: 24GB |
| 训练框架 | PyTorch + Hugging Face |
| 优化策略 | LoRA (16-bit) |
| 有效批次 | 128 (V5.2口径) |

## 3. 超参数配置

### 3.1 6GB配置 (configs/train_6gb.yaml)

```yaml
# Qwen2.5-1.5B SFT训练配置 - 6GB显存版本
experiment:
  name: "qwen-1.5b-sft-6gb"
  type: "exp1_direct_sft"
  seed: 42

model:
  name: "Qwen2.5-1.5B-Instruct"
  path: "D:/30_keyan/GeoKD-SR/models/Qwen2.5-1.5B-Instruct"
  use_lora: true
  lora:
    r: 8
    alpha: 16
    dropout: 0.05
    target_modules: ["q_proj", "k_proj", "v_proj", "o_proj"]

training:
  learning_rate: 5e-5            # LoRA推荐值
  batch_size: 1                  # 6GB限制
  gradient_accumulation_steps: 128  # effective_batch=128
  num_epochs: 3
  warmup_ratio: 0.1
  weight_decay: 0.01
  max_grad_norm: 1.0
  lr_scheduler_type: "cosine"

optimization:
  max_length: 1024               # 降低以节省显存
  gradient_checkpointing: true
  mixed_precision: "fp16"        # 6GB不支持bf16

data:
  train_file: "data/splits/train.jsonl"
  dev_file: "data/splits/dev.jsonl"

output:
  base_dir: "exp/exp0/qwen-1.5B-sft/outputs"
  logging_dir: "exp/exp0/qwen-1.5B-sft/logs"
  checkpoint_dir: "exp/exp0/qwen-1.5B-sft/checkpoints"
```

### 3.2 24GB配置 (configs/train_24gb.yaml)

```yaml
# Qwen2.5-1.5B SFT训练配置 - 24GB显存版本
experiment:
  name: "qwen-1.5b-sft-24gb"
  type: "exp1_direct_sft"
  seeds: [42, 123, 456, 789, 1024]  # 5次实验

model:
  name: "Qwen2.5-1.5B-Instruct"
  path: "/mnt/data/models/Qwen2.5-1.5B-Instruct"
  use_lora: true
  lora:
    r: 16
    alpha: 32
    dropout: 0.05
    target_modules: ["q_proj", "k_proj", "v_proj", "o_proj"]

training:
  learning_rate: 5e-5            # LoRA推荐值
  batch_size: 8
  gradient_accumulation_steps: 16   # effective_batch=128
  num_epochs: 3
  warmup_ratio: 0.1
  weight_decay: 0.01
  max_grad_norm: 1.0
  lr_scheduler_type: "cosine"

optimization:
  max_length: 2048
  gradient_checkpointing: true
  mixed_precision: "bf16"

data:
  train_file: "/mnt/data/GeoKD-SR/data/splits/train.jsonl"
  dev_file: "/mnt/data/GeoKD-SR/data/splits/dev.jsonl"

output:
  base_dir: "/mnt/output/qwen-1.5B-sft"
  logging_dir: "/mnt/output/qwen-1.5B-sft/logs"
  checkpoint_dir: "/mnt/output/qwen-1.5B-sft/checkpoints"
```

### 3.3 配置对比

| 参数 | 6GB配置 | 24GB配置 | 说明 |
|------|---------|----------|------|
| learning_rate | 5e-5 | 5e-5 | LoRA推荐值 |
| batch_size | 1 | 8 | 适配显存 |
| gradient_accumulation | 128 | 16 | effective_batch=128 |
| max_length | 1024 | 2048 | 适配显存 |
| mixed_precision | fp16 | bf16 | 硬件支持 |
| LoRA r | 8 | 16 | 参数量 |
| warmup_ratio | 0.1 | 0.1 | 预热10% |
| lr_scheduler | cosine | cosine | 余弦退火 |

## 4. 数据处理

### 4.1 数据格式转换

**原始格式** (splits/train.jsonl):
```json
{
  "id": "geosr_directional_01195",
  "question": "从大连看，孝感位于哪个方向？",
  "answer": "西南方向。",
  "spatial_relation_type": "directional",
  "reasoning_chain": [...]
}
```

**ChatML训练格式**:
```json
{
  "messages": [
    {"role": "system", "content": "你是一个地理空间推理专家..."},
    {"role": "user", "content": "从大连看，孝感位于哪个方向？"},
    {"role": "assistant", "content": "西南方向。"}
  ]
}
```

### 4.2 Label构造（V5.2公平性核心）

- system段 → labels = -100
- user段 → labels = -100
- assistant段 → labels = token_ids（仅此处计算损失）

## 5. 训练流程

```
┌─────────────────────────────────────────────────────────────┐
│                     训练流程                                  │
├─────────────────────────────────────────────────────────────┤
│  1. 加载配置 (train_6gb.yaml 或 train_24gb.yaml)             │
│  2. 加载模型 + Tokenizer                                      │
│     - 使用ChatML template (apply_chat_template)              │
│     - 加载LoRA配置 (PEFT)                                     │
│  3. 加载数据集                                                │
│     - 选择 splits 或 split_coords                            │
│     - 转换为ChatML格式                                        │
│  4. 配置TrainingArguments                                    │
│  5. 启动SFTTrainer训练                                        │
│  6. 保存checkpoints                                          │
│  7. 运行评测                                                  │
└─────────────────────────────────────────────────────────────┘
```
