#### S-PERS-03: 消融实验对比逻辑存在潜在漏洞

**位置**: 第3.2节 验证目标、第4.7.6节 消融实验对比说明

**问题描述**:

文档声称通过消融实验可以验证每个组件的独立贡献，但存在以下逻辑漏洞：

**漏洞1: 组件组合的协同效应未分离**
```
Exp9 = B2 + C1 + C2 + C3 + C4 + C5 + C6
Exp9 vs Exp2 的差异 = C1 + C2 + C3 + C4 + C5 + C6 + 协同效应

问题：无法区分单个组件的贡献和组件间的协同效应
```

**漏洞2: Exp3a设计存在概念混淆**
```
Exp3a: B2 + C1 (等权重 [1.0, 1.0, 1.0, 1.0])
Exp3:  B2 + C1 (可学习权重)

问题：Exp3a实际上仍然是"引入空间关系类型"，只是权重不同
      真正的对照应该是"不区分空间关系类型"
```

**漏洞3: 消融实验无法完全控制变量**
```
各组件的输入要求不同：
- C1需要spatial_relation_type
- C2需要reasoning_chain
- C5需要entities, spatial_tokens

问题：使用不同字段→实际上改变了输入数据→违反"控制变量"原则
```

**影响分析**:
- 审稿人可能质疑消融实验设计的有效性
- 无法真正证明"每个组件的独立贡献"
- 可能要求重新设计实验

**改进建议**:
```markdown
【建议修改】（第3.2节、第4.7.6节）

**修改1: 明确消融实验的局限性**
在第3.2节添加声明：
```
本研究采用组件级消融实验，旨在验证各组件的有效性。
需要注意的是，由于各组件的输入要求不同，完全严格的变量控制在技术上不可行。
我们采用"统一数据+选择性使用"策略，将数据差异降至最低。
```

**修改2: 重新设计Exp3a**
```
当前设计：
Exp3a: B2 + C1 (等权重)
Exp3:  B2 + C1 (可学习权重)

建议改为：
Exp3a: B2 + 基础蒸馏（不区分关系类型）
Exp3:  B2 + C1 (区分关系类型，等权重)
Exp3b: B2 + C1 (区分关系类型，可学习权重)

这样可以分离三个效应：
1. 是否区分关系类型（Exp3a vs Exp2）
2. 是否使用权重（Exp3 vs Exp3a）
3. 权重是否可学习（Exp3b vs Exp3）
```

**修改3: 添加协同效应分析**
在实验设计部分添加：
```
实验Exp9（完整方法）的性能提升来源于：
- 单个组件的直接贡献（通过Exp3-Exp8评估）
- 组件间的协同效应（通过组合实验评估）

为分离协同效应，我们设计以下组合实验：
- Exp9a: B2 + C1 + C2（测试C1和C2的协同）
- Exp9b: B2 + C1 + C5（测试C1和C5的协同）
- ...

如果Exp9a的性能 > Exp3 + Exp4 - Exp2，则证明存在正向协同效应。
```
```

def detect_data_leakage(predictions, references, n=4):
    """检测预测与参考的n-gram重叠度"""
    from collections import Counter
    from nltk import ngrams

    pred_ngrams = Counter()
    ref_ngrams = Counter()

    for pred in predictions:
        pred_ngrams.update(ngrams(pred.split(), n))

    for ref in references:
        ref_ngrams.update(ngrams(ref.split(), n))

    overlap = sum((pred_ngrams & ref_ngrams).values())
    total = sum(pred_ngrams.values())
    overlap_ratio = overlap / total if total > 0 else 0

    return overlap_ratio
