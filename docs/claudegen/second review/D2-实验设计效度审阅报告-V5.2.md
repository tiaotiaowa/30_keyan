#### 🟡 中等问题D2-M3：地理实体的区域覆盖不均衡

**位置**：
- 第631-644行：地理实体库构建
- 第636-644行：均衡采样策略

**问题描述**：
V5.2声称"按地理区域均衡采样"，但实际实现可能存在问题：
```python
# 第636-644行
**均衡采样策略**:
- 每个区域采样数量与该区域实体总数成正比
- 确保每种空间关系类型的问题覆盖所有区域
```

**科学问题**：
1. **"与实体总数成正比"≠均衡**：如果东部实体多于西部，按比例采样会导致东部问题更多
2. **区域定义不明确**：如何定义"华北、华东、华南、西部"？按省份？按经济区？
3. **验证缺失**：没有说明如何验证数据集的区域均衡性

**影响分析**：
- 中等影响：数据集可能存在地理区域偏差
- 泛化性风险：模型可能在东部地理问题上表现更好
- 外部效度受限：结果不能泛化到所有中国区域

**修改建议**：
```markdown
# 修改第636-644行
**均衡采样策略**:
1. 区域定义：按国家统计局区域划分（华北、东北、华东、华中、华南、西南、西北）
2. 采样配额：每个区域的基础配额=总样本量/7≈1143条
3. 实体内采样：在每个区域内按实体重要性（省会、地级市）分层采样
4. 偏差验证：报告每个区域的样本量分布

**验证指标**:
- 区域样本变异系数(CV) < 0.2
- 每个区域至少800条样本
```
