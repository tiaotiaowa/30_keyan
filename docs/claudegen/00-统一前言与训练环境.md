> - V6.0 更贴近“可跑通复现”
> - V5.2 更贴近“论文口径与统计规范”

### 1.4 标签与掩码策略（公平性核心）
| 项目 | 统一口径 | V5.2 依据 |
|-----|---------|-----------|
| labels 构造 | system+user 段 = -100，仅 assistant 段计算 CE | §4.7.4 公平性保证机制 |
| valid_mask（蒸馏） | `(labels != -100) & (attention_mask == 1)` | §4.7.4 |
| 监督范围 | **禁止对 prompt 段计算蒸馏损失** | §4.7.4 |

> **违反此原则的后果**：KL 对 prompt 计算 KD 会破坏公平对比，不同实验的“真实蒸馏强度”将不可比。

### 1.5 评测生成参数（所有实验一致）
| 项目 | 统一口径 | V5.2 依据 |
|-----|---------|-----------|
| temperature | 0.1 | §7.5 评测 pipeline |
| top_p | 0.9 | §7.5 |
| do_sample | True | §7.5 |
| max_new_tokens | 512 | §7.5 |

### 1.6 评测协议（引用统一协议文档）
详见 `docs/claudegen/GeoKD-SR-实验实现与评测体系统一设计_V5.2细化_20260311.md`：
- **主评测集**：final test split（从 `final_1_final.jsonl` 导出）
- **输出格式**：强制 JSON 块（`relation_type`, `final_answer`, 对应字段）
- **评分规则**：结构化为主（方向归一化、拓扑映射、距离容差）
- **分层统计**：按 `spatial_relation_type` / `difficulty` / `topology_subtype`
- **错误 taxonomy**：`format_error`, `wrong_direction`, `wrong_topology`, `wrong_distance_value` 等

## 3. Seeds 与重复次数（统计规范）

### 3.1 Seeds 集合（V5.2 论文口径）
```
seeds = [42, 123, 456, 789, 1024]
```
**V5.2 依据**：§6.1.1

### 3.2 重复次数
- **论文级统计**：每个实验运行 **5 次**（上述 5 seeds）
- **报告格式**：均值 ± 标准差（mean ± std）
- **显著性检验**：Shapiro 正态性检查 → 配对 t 检验或 Wilcoxon → Cohen's d / Cliff's Delta → Holm-Bonferroni 校正

**V5.2 依据**：§6.1.1 - §6.2

> **注**：V6.0 执行手册建议每个实验运行 3 次（seeds 42/43/44），但**本口径以 V5.2 的 5 次为准**。

## 4. 目录结构与产物协议

### 4.1 Checkpoint 保存路径
```
checkpoints/{method}/{seed}/
```
**V5.2 依据**：§6.3

- `{method}` = exp01, exp02, ..., exp09
- `{seed}` = 42, 123, 456, 789, 1024
- 命名规则：`checkpoint-epoch{N}-acc{X.XX}.pt`

### 4.2 日志与结果目录
```
logs/{method}/{seed}/          # train.log, eval.log
results/{method}/{seed}/       # metrics.json, predictions.jsonl, report.md
```

### 4.3 防覆盖机制（可选）
在上述目录下引入 `run_id` 层级：
```
checkpoints/{method}/{seed}/{run_id}/
```
其中 `run_id` 推荐格式：`{timestamp}_{gitShort}_{method}_{seed}`

## 7. 与 V6.0 执行手册的主要差异

| 项目 | V5.2（本口径） | V6.0（执行手册） |
|-----|---------------|-----------------|
| seeds | [42,123,456,789,1024]（5次） | [42,43,44]（3次） |
| learning_rate | 1e-4 | 2e-5 |

| gradient_accumulation | 16 | 1 |
| 有效 batch | 128 | 16 |
| warmup_ratio | 0.1 | 0.03 |
| epochs | 3 | 3 |
| **目的定位** | 论文口径与统计规范 | 可跑通复现 |

**建议**：实际训练时，如受硬件限制，可按 V6.0 调整 batch/accum，但需保持 effective batch 相近，并在报告时注明调整原因。
