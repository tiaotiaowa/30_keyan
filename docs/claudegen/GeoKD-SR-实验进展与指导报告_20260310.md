## 三、数据质量与异常现状
### 3.1 异常概况（最终版本）
- 总样本：**11,656**
- 异常记录：**914（7.84%）**
- 异常类型：
  - **answer_mismatch**（占比最高）：答案与推理链 final_answer 语义冲突或表达不一致
  - **entity_not_in_entities**：reasoning_chain 中实体未出现在 entities 列表

### 3.2 抽样发现
- 存在**语义矛盾**示例：
  - answer：不存在包含关系
  - final_answer：集安位于上海市内
- 说明当前一致性检测规则能捕获明显矛盾，但仍需语义判定以过滤误报。

### 3.3 建议修复策略（已确认）
- **answer_mismatch**：语义一致即通过；仅保留明显矛盾样本作为异常
- **entity_not_in_entities**：通过**实体别名映射 + 补齐 entities** 修复

### 3.4 验收阈值（建议执行）
- 字段完整率：100%
- 语义一致率（answer vs final_answer）：≥98%
- 实体一致率（entities_involved ⊆ entities）：≥99%
- 坐标合法率：100%
- 关系类型与难度分布偏差：≤2%

## 四、与实验设计对齐要点
### 4.1 消融矩阵
- Exp1: Direct‑SFT（无蒸馏）
- Exp2: Standard‑KD（Forward KL）
- Exp3a: B2 + C1（等权）
- Exp3: B2 + C1（可学习权重）
- Exp4: B2 + C2（思维链蒸馏）
- Exp5: B2 + C3（逆向KL）
- Exp6: B2 + C4（自蒸馏）
- Exp7: B2 + C5（空间注意力蒸馏）
- Exp8: B2 + C6（渐进式蒸馏）
- Exp9: 完整 GeoKD‑SR

### 4.2 评测与统计规范
- 主要指标：**RA 推理准确率**（步骤/答案/完整性/清晰度加权）
- 次要指标：空间关系 F1、知识保留率
- 辅助指标：BLEU、ROUGE‑L、Perplexity
- 统计要求：5 次重复实验 + 显著性检验（t/Wilcoxon）+ 效应量 + Holm‑Bonferroni 校正

### 4.3 数据规模与分层策略（已确认）
- 总计保留 **10,000 条**
- 按 **8:1:1** 分层抽样划分 train/dev/test
- GeoSR‑Bench 从 test 集按关系类型×难度分层抽样 3,000 题
