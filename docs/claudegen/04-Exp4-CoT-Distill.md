## 2. 理论依据

### 2.1 KD/LLM 蒸馏角度

#### 2.1.1 思维链蒸馏的动机
**传统 KD 的局限**：仅监督最终答案，未监督推理过程
- **问题**：学生可能学会“猜答案”而非“学会推理”
- **风险**：在复杂任务（如 composite 推理）中泛化差

**CoT Distillation 的解决**：让学生学习教师的推理链
- **显式监督**：每个推理步骤都有 KL 监督
- **结构化知识**：推理链编码了问题→结论的映射

#### 2.1.2 相关工作
- **Wei et al., 2022**：Chain-of-Thought Prompting
- **ACL 2023 推理蒸馏工作**：蒸馏推理过程而非仅答案

**V5.2 依据**：§4.7.4 "C2: 思维链蒸馏…思想：蒸馏推理过程，而非仅答案"

### 2.2 地理空间推理角度

#### 2.2.1 Geo 任务中的推理链价值

**推理链示例**（directional）：
```
步骤1：识别实体 A = "北京"，B = "天津"
步骤2：获取坐标 A(116.4, 39.9), B(117.2, 39.1)
步骤3：计算经度差 = 117.2 - 116.4 = 0.8°（东）
步骤4：计算纬度差 = 39.1 - 39.9 = -0.8°（南）
步骤5：综合判断：东南方向
结论：北京在天津的西北方向，天津在北京的东南方向
```

**推理链的 Geo 价值**：
1. **中间步骤可验证**：坐标计算、方向判断可分别监督
2. **错误可追溯**：若结论错，可定位到哪一步错
3. **能力迁移**：学会推理步骤可泛化到新问题

#### 2.2.2 不同关系类型的推理链特点

| 类型 | 推理链特点 | 关键步骤 |
|-----|----------|---------|
| directional | 坐标→方向计算 | 识别实体、计算差值、方向映射 |
| topological | 包含关系判断 | 九交模型计算、拓扑分类 |
| metric | 距离计算 | 坐标投影、距离公式、单位转换 |
| composite | 多步骤组合 | 方向+距离综合推理 |

### 3.3 损失函数

#### 3.3.1 总损失
```
L_SCOT = α × L_chain + (1-α) × L_answer
```
- **α = 0.6**：推理链权重更高
- **T = 2.0**：温度参数

#### 3.3.2 L_chain（推理链损失）
```
L_chain = (1/n) × Σ_{i=1}^{n} KL(P_T^{(i)} || P_S^{(i)})
```
- **n**：推理链步骤数（V5.2 数据集中 n=5）
- **P_T^{(i)}, P_S^{(i)}**：第 i 步的教师/学生分布
- **归一化**：1/n 防止链长影响损失规模

#### 3.3.3 L_answer（答案损失）
```
L_answer = KL(P_T^{final} || P_S^{final})
```
- **final**：最终答案步骤

**V5.2 依据**：§4.7.4 "L_SCOT = α × L_chain + (1-α) × L_answer…α=0.6…T=2.0"

### 3.4 训练流程要点

#### 3.4.1 推理链分割
```python
# 推理链格式（数据集中）
reasoning_chain = [
    {"step": 1, "content": "识别实体..."},
    {"step": 2, "content": "获取坐标..."},
    {"step": 3, "content": "计算..."},
    {"step": 4, "content": "综合判断..."},
    {"step": 5, "content": "结论..."}
]

# 转为监督文本
chain_text = "\n".join([f"步骤{s['step']}：{s['content']}" for s in reasoning_chain])
target_text = f"{chain_text}\n\n最终答案：{answer}"
```

#### 3.4.2 分步 KL 计算
```python
# 前向
student_logits = student(input_ids)  # [batch, seq_len, vocab]
with torch.no_grad():
    teacher_logits = teacher(input_ids)

# 分割推理链（简化：按 token 分割）
# 实际实现需要按步骤边界分割
chain_masks = get_chain_masks(reasoning_chain, tokenizer)

# 计算每步 KL
loss_chain = 0
for i, mask in enumerate(chain_masks):
    kl_i = KL_div(teacher_logits[mask], student_logits[mask])
    loss_chain += kl_i
loss_chain = loss_chain / len(chain_masks)  # 归一

# 答案 KL
loss_answer = KL_div(teacher_logits[answer_mask], student_logits[answer_mask])

# 总损失
loss = α × loss_chain + (1-α) × loss_answer
```

#### 3.4.3 关键实现细节
- **步骤分割**：需要准确定位每步的 token 边界
- **mask 对齐**：教师与学生使用相同的步骤边界
- **显存优化**：推理链较长，可能需要梯度检查点

## 5. 诊断与失败模式

### 5.1 常见陷阱

#### 5.1.1 “推理链模板化”
- **陷阱**：学生学会模板化推理（“步骤1：识别实体…”）但不提升正确率
- **诊断**：推理链看起来正确，但结论仍错
- **后果**：overfitting 推理格式，未真正学会推理

#### 5.1.2 “推理链质量不稳定”
- **陷阱**：教师推理链有噪声（如错误步骤）
- **诊断**：抽样检查数据集推理链
- **后果**：学生学到错误的推理模式

#### 5.1.3 “步骤分割错误”
- **陷阱**：代码中步骤边界定位错误
- **诊断**：打印分割结果，人工验证
- **后果**：KL 计算范围错误

### 5.2 性能诊断

| 现象 | 可能原因 | 建议排查 |
|-----|---------|---------|
| Exp4 ≈ Exp2 | 推理链无额外价值 | 检查推理链质量 |
| Exp4 < Exp2 | 推理链引入噪声 | 检查 α、数据质量 |
| Exp4 >> Exp2 | 推理链显著改善 | 分析哪类任务受益最大 |

### 5.3 不同关系类型的效果差异

| 类型 | CoT 预期效果 | 原因 |
|-----|------------|------|
| directional | 中等提升 | 推理链显式化方向计算 |
| topological | 较大提升 | 九交模型推理步骤多 |
| metric | 较大提升 | 数值计算步骤清晰 |
| composite | 最大提升 | 多步骤组合，推理链价值高 |
