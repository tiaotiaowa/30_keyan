# 第一部分：研究背景与问题定义

## 1.1 研究背景

### 地理人工智能的发展趋势

将大型语言模型（LLMs）集成到地理信息系统（GIS）中，标志着地理人工智能（GeoAI）领域的一次革命性转变。随着空间数据体量和复杂性的持续增长，LLMs为空间分析自动化、自然语言交互实现以及GIS环境中的分析推理增强提供了新范式。

然而，当前LLM4GIS研究大多依赖于云端大规模模型，其在**端侧设备**上的部署面临严峻的算力与能耗约束。现有端侧方案多采用单一压缩模型，在需要深度**地图认知、空间关系推理与地学知识融合**的任务上性能受限。

### 2025年关键发现

**重要发现1：大模型的"虚假空间智能"问题**
- 2025年OSI-Bench揭示：大模型在空间推理基准上的高分可能是"背答案"而非真正习得空间推理能力
- 训练数据与测试数据同源问题严重

**重要发现2：知识蒸馏在推理任务上的有效性**
- DeepSeek-R1（2025）验证：蒸馏在推理能力迁移上优于强化学习
- 斯坦福团队：思维链蒸馏可有效迁移推理能力

### 研究空白

| 空白领域 | 现状 | 机会 |
|---------|------|------|
| GIS领域知识蒸馏 | 仅2篇论文，均聚焦遥感图像 | **空间推理能力蒸馏**尚未被研究 |
| 空间认知蒸馏 | 无系统研究 | 首次探索方向/拓扑/距离推理的蒸馏机制 |
| 地理幻觉抑制 | 无知识图谱引导方法 | 知识图谱校验的蒸馏新范式 |

## 1.2 研究问题

**核心研究问题**：如何通过知识蒸馏将大模型的地理空间关系推理能力高效迁移到轻量化模型？

**子问题**：
1. 如何设计针对空间关系类型的蒸馏损失函数？
2. 不同类型的空间关系（方向/拓扑/度量）在蒸馏过程中有何差异？
3. 如何评估轻量化模型的地理空间推理能力？

```python
# 逆向KL蒸馏（借鉴DeepSeek-R1）
def reverse_kl_loss(student_logits, teacher_logits, temperature=1.0):
    """
    逆向KL散度：KL(p_student || p_teacher)
    优点：避免模式崩溃，保持生成多样性
    """
    student_probs = F.softmax(student_logits / temperature, dim=-1)
    teacher_probs = F.softmax(teacher_logits / temperature, dim=-1)
    rkl = torch.sum(student_probs * (torch.log(student_probs) - torch.log(teacher_probs)), dim=-1)
    return rkl.mean()

# 选择性Token蒸馏（借鉴AdaSPEC）
def selective_distillation(teacher_output, student_output, difficulty_scores):
    """
    选择性蒸馏：过滤难以学习的Token
    """
    mask = difficulty_scores < threshold  # 选择"易学习"的Token
    return kl_loss(teacher_output[mask], student_output[mask])
```

### 核心创新：空间关系类型感知的蒸馏损失

```python
class SpatialRelationAwareDistillationLoss(nn.Module):
    """
    空间关系类型感知的蒸馏损失

    核心思想：不同类型的空间关系在蒸馏过程中应该有不同的权重
    """

    def __init__(self, temperature=2.0):
        super().__init__()
        self.temperature = temperature

        # 空间关系类型权重（可调参）
        self.relation_weights = {
            'directional': 1.5,  # 方向关系权重更高
            'topological': 1.3,  # 拓扑关系
            'metric': 1.0,       # 度量关系
            'order': 1.0,        # 序关系
            'general': 1.0       # 一般文本
        }

    def identify_relation_type(self, text):
        """
        识别文本中的空间关系类型
        使用规则+词表的方法
        """
        # 方向词表
        direction_words = ['东', '西', '南', '北', '东北', '西南', '东南', '西北',
                          '左', '右', '前', '后', '上', '下']
        # 拓扑词表
        topology_words = ['包含', '相邻', '相交', '位于', '穿过', '接壤', '内部']
        # 度量词表
        metric_words = ['距离', '远', '近', '公里', '米', '面积', '长度']

        for word in direction_words:
            if word in text:
                return 'directional'
        for word in topology_words:
            if word in text:
                return 'topological'
        for word in metric_words:
            if word in text:
                return 'metric'

        return 'general'

    def forward(self, teacher_logits, student_logits, input_texts):
        """
        计算空间关系感知的蒸馏损失

        Args:
            teacher_logits: 教师模型的logits [batch_size, seq_len, vocab_size]
            student_logits: 学生模型的logits [batch_size, seq_len, vocab_size]
            input_texts: 输入文本列表 [batch_size]
        """
        total_loss = 0

        for i, text in enumerate(input_texts):
            # 识别空间关系类型
            rel_type = self.identify_relation_type(text)
            weight = self.relation_weights[rel_type]

            # 计算KL散度
            teacher_probs = F.softmax(teacher_logits[i] / self.temperature, dim=-1)
            student_log_probs = F.log_softmax(student_logits[i] / self.temperature, dim=-1)

            kl = F.kl_div(student_log_probs, teacher_probs, reduction='batchmean')

            # 加权损失
            total_loss += weight * kl

        return total_loss / len(input_texts)
```

### 辅助技术

```python
# 1. 逆向KL蒸馏（借鉴DeepSeek-R1）
def reverse_kl_component(student_logits, teacher_logits, alpha=0.3):
    """
    逆向KL作为辅助损失，避免模式崩溃
    """
    student_probs = F.softmax(student_logits, dim=-1)
    teacher_probs = F.softmax(teacher_logits, dim=-1)
    rkl = torch.sum(student_probs * (torch.log(student_probs + 1e-10) - torch.log(teacher_probs + 1e-10)), dim=-1)
    return alpha * rkl.mean()

# 2. 选择性Token蒸馏（借鉴AdaSPEC）
def selective_token_filter(teacher_logits, student_logits, token_difficulties, threshold=0.7):
    """
    过滤难以学习的Token，聚焦"易学习"部分
    """
    mask = token_difficulties < threshold
    if mask.sum() > 0:
        return kl_loss(teacher_logits[mask], student_logits[mask])
    return 0

# 3. 总损失函数
class TotalDistillationLoss(nn.Module):
    def __init__(self):
        super().__init__()
        self.spatial_loss = SpatialRelationAwareDistillationLoss()
        self.reverse_kl_weight = 0.3
        self.task_loss_weight = 0.5

    def forward(self, teacher_logits, student_logits, input_texts, labels):
        # 空间关系感知损失
        spatial_loss = self.spatial_loss(teacher_logits, student_logits, input_texts)

        # 逆向KL损失
        rkl_loss = reverse_kl_component(student_logits, teacher_logits, self.reverse_kl_weight)

        # 任务特定损失（如问答）
        task_loss = F.cross_entropy(student_logits.view(-1, student_logits.size(-1)), labels.view(-1))

        # 总损失
        total = spatial_loss + rkl_loss + self.task_loss_weight * task_loss
        return total
```

## A.1 完整蒸馏训练流程

```python
import torch
import torch.nn as nn
import torch.nn.functional as F
from transformers import AutoModelForCausalLM, AutoTokenizer

class GeoKDSRTrainer:
    def __init__(self, teacher_model, student_model, tokenizer):
        self.teacher = teacher_model
        self.student = student_model
        self.tokenizer = tokenizer

        # 冻结教师模型
        for param in self.teacher.parameters():
            param.requires_grad = False

        # 初始化损失函数
        self.spatial_loss = SpatialRelationAwareDistillationLoss()
        self.temperature = 2.0

    def train_step(self, batch):
        # 获取教师输出
        with torch.no_grad():
            teacher_outputs = self.teacher(
                input_ids=batch['input_ids'],
                attention_mask=batch['attention_mask']
            )
            teacher_logits = teacher_outputs.logits

        # 获取学生输出
        student_outputs = self.student(
            input_ids=batch['input_ids'],
            attention_mask=batch['attention_mask']
        )
        student_logits = student_outputs.logits

        # 计算蒸馏损失
        texts = self.tokenizer.batch_decode(batch['input_ids'])
        distill_loss = self.spatial_loss(teacher_logits, student_logits, texts)

        # 计算任务损失
        task_loss = F.cross_entropy(
            student_logits.view(-1, student_logits.size(-1)),
            batch['labels'].view(-1)
        )

        # 总损失
        total_loss = distill_loss + 0.5 * task_loss

        return total_loss

    def train(self, train_loader, epochs, lr=1e-5):
        optimizer = torch.optim.AdamW(self.student.parameters(), lr=lr)

        for epoch in range(epochs):
            for batch in train_loader:
                loss = self.train_step(batch)

                loss.backward()
                optimizer.step()
                optimizer.zero_grad()

            print(f"Epoch {epoch}: Loss = {loss.item()}")
```
