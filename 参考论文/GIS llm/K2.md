## 摘要

大型语言模型在自然语言处理的通用领域取得了巨大成功。在本文中，我们将大型语言模型带入地球科学领域，旨在推进该领域的研究与应用。为此，我们推出了地球科学领域首个大型语言模型 K2，以及一系列为促进地球科学领域大型语言模型研究而开发的配套资源。例如，我们精心整理了一个地球科学指令微调数据集 GeoSignal，旨在使模型的响应与地球科学相关的用户查询对齐。此外，我们建立了一个地球科学领域的首个基准测试集 GeoBench，用于在地球科学背景下评估大型语言模型。在这项工作中，我们尝试了一套完整的方案，将预训练的通用领域大语言模型适配到地球科学领域。具体来说，我们在包含超过100万篇地球科学文献的55亿词元的地球科学文本语料库上，对 LLaMA-7B 模型进行了进一步训练，并利用 GeoSignal 的监督数据对模型进行微调。此外，我们分享了一个即使在人力稀缺的情况下，也能高效收集领域特定数据并构建领域监督数据的流程。同时，我们为 K2 配备了使用工具的能力，使其成为一个初级的地球科学助手。在 GeoBench 上进行的实验证明了我们的方法和数据集在地球科学知识理解与应用方面的有效性。我们在 https://github.com/davendw49/k2 开源了所有的训练数据和 K2 模型检查点。

K2 的训练包括两个阶段，预训练阶段和指令微调阶段，如图 1 所示。在预训练期间，我们在从地球科学论文预处理得到的地球科学文本语料库上继续预训练 LLaMA-7B 模型。然后我们执行指令微调 [5, 26, 41]，在此阶段我们进一步训练模型以遵循人类指令。为此，我们精心整理了 GeoSignal，一个通过将来自 8 个不同地球科学自然语言处理任务（如关系抽取、实体识别、分类和摘要）的示例与提示词统一起来创建的指令微调数据集。我们还构建了 GeoBench，一个包含从全国硕士研究生入学考试地球科学科目和美国大学先修课程考试（地质学、地理学和环境科学）收集的超过1500道客观题和939道主观题的评估数据集。GeoBench 用于跟踪地球科学语言模型的进展并推动其发展。通过我们在数据收集和训练方面的协同努力，最终得到的 K2 模型是一个基础语言模型，可用于设计多种地球科学应用，从而使地球科学研究人员和从业者受益[31]。为了例证这一点，我们训练 K2 通过学习使用工具来掌握地球科学学术搜索引擎的使用，同时，通过思维链引导 K2 进行地球科学相关知识点的关系预测并生成新想法。因此，K2 在地球科学知识挖掘和研究辅助方面展现了其潜力。

我们的贡献可归纳如下：

- 我们推出了 K2，一个地球科学领域的基础语言模型。K2 可以回答地球科学问题，通过其在地球科学领域的专业性，使用合适的提示词遵循地球科学家的指令，并且具备使用工具作为扩展的能力。
- 我们构建了 GeoSignal，首个地球科学监督指令数据。
- 为了在地球科学任务上评估 K2 以及后续的地球科学语言模型，我们建立了 GeoBench，首个地球科学领域的自然语言处理任务基准。
- 以地球科学为例，我们建立了一套构建领域文本语料库、领域监督指令数据并探索训练领域特定 LLM 方案的范式。与类似尺寸的基线模型相比，K2 在主观和客观地球科学任务上均表现更优，包括参加考试和进行知识推理。
- 最后，我们在 https://github.com/davendw49/k2 开源了所有代码、K2 权重、GeoSignal 和 GeoBench。

## 2 相关工作

基础语言模型。自 ChatGPT [35] 出现以来，涌现出大量用作基础模型来解决现实问题的大型语言模型。由于仅提供在线演示和 API 的模型（如 ChatGPT、GPT-4 [36] 和文心一言）不适合且不方便进一步预训练和开发，因此像 CodeGen [34]、LLaMA [47]、GLM [54] 这样的开源模型成为了许多其他指令微调 LLM（如 Alpaca [44]、Baize [52]、Vicuna [4]、Koala [15] 和 Dolly [7]）的基础模型。同时，随着使用外部模块和信息增强基础模型的迅速兴起，工具学习也因 LLM 而受到关注。

领域语言模型。大型语言模型成为解决许多其他领域问题的基础模型。在生命科学领域，有 Med-PaLM [43]、MedGPT [22]、BioGPT [28] 和 Bio-Megatron [42]。大型语言模型在生物医学领域非常有用且可靠[48]。在自然科学领域，Geographic-BERT [25]、MGeo [11]、PK-Chat [9]、ERNIE-GeoL [20] 和 GeoBERT [10] 是地理学和地质学领域的典型案例[32]，而 MatSciBERT [17] 则是材料科学领域的例子。在学术场景中，SciBERT [2] 和 Galactica [45] 是两个例子。

LLM 的参数高效微调。传统的微调需要更新 LLM 中的所有参数，随着模型遵循缩放定律[21]而增长，这导致效率低下并留下巨大的碳足迹。软提示微调[23]冻结了语言模型以执行特定的下游任务。前缀微调[24]从语言模型的提示中汲取灵感，允许后续词元像对待“虚拟词元”一样关注此前缀。此外，适配器[18]使原始网络的参数保持固定，实现了高度的参数共享，而 LoRA [19]将权重的更新视为两个可训练的低秩矩阵相乘的结果。

## 3 数据收集与整理

为了训练 K2，我们从各种资源收集了地球科学文本语料库和面向地球科学的数据。然后，我们将这些数据重新构建成信号，并建立了指令微调数据集 GeoSignal。这些宝贵的信息可用于学习地球科学任务的知识，并指导模型与人类和专家对齐。此外，我们开发了 GeoBench 来比较专注于地球科学的语言模型。

### 3.1 预训练数据

在这项工作中，我们用于在 LLaMA-7B 上进行继续预训练的文本语料库包含 55 亿词元，包括与地球科学相关的维基百科页面、地球科学论文摘要以及选自地球科学领域高质量期刊并由 GAKG [8] 主要收集的开放获取地球科学论文。

#### 3.1.1 地球科学文本语料库收集。

#### 3.1.2 文本语料库预处理。

PDF 解析。我们基于 GROBID 库 [1] 构建了一个自动化的 PDF 解析工具包。我们使用 Markdown 作为语料库中所有论文的格式，以保持可读性和一致性。最后，我们使用正则表达式和基于规则的脚本清理数据，删除妨碍阅读的文本、乱码和不纯数据。该脚本将在最终稿完成后发布，目前正由 DeepShovel [55] 使用。

词元化。词元化是文本语料库设计的重要组成部分。为了使语言模型理解学术论文，我们对不同的模态使用专门的词元，如下所示，示例展示在图 2 中。

- 插图：我们使用特殊词元 [START_FIGURE] 和 [END_FIGURE] 来标注论文插图的标题。
- 表格：使用两个特殊词元 [START_TABLE] 和 [END_TABLE] 来定位文章中表格的位置。在此过程中，我们将 PDF 中的表格转换为 Markdown 格式。
- 引用：我们使用特殊词元 [START_REF] 和 [END_REF] 来标注引用。
- 公式：对于数学内容或公式，我们通过正则表达式和基于规则的方法过滤和清理从 PDF 解析出的不规则公式。此外，我们使用特殊词元 [START_FORMULA] 和 [END_FORMULA] 来捕获它们。

3.2.1 与人类对齐。在这一部分，我们收集和汇总了几个构建良好的监督数据集，包括自指令、人工标注和工具相关数据。

- Alpaca-GPT4：Alpaca-GPT4 是一个通过名为 Self-Instruct [50] 的技术生成的指令遵循数据集，所有样本的形式均为 <指令，输入，输出>，我们选择遵循此格式。
- Dolly-15k：databricks-dolly-15k [7] 是一个由数千名 Databricks 员工生成的指令遵循记录的开源数据集，包括头脑风暴、分类、封闭式问答、生成、信息抽取、开放式问答和摘要。我们将它们全部整理成 <指令，输入，输出> 格式。
- Natural Instruction：Natural Instruction [33] 维护了许多任务及其自然语言定义/指令。其 v1.x 数据集包含 61 个任务。v2.x 数据集包含超过 1500 个任务。我们从 v2.x 数据集中精心挑选了客观任务，并将其整理成 <指令，输入，输出> 格式。
- AI2 Reasoning Challenge：AI2 Reasoning Challenge (ARC) [6] 是一个包含 7,787 道真实的小学水平多项选择科学问题的数据集。由于其结构良好，我们随机采样并将其整理成 <指令，输入，输出> 格式。
- 工具指令数据：参考 ToolBench [38]，我们手动整理了一个包含 2k 条指令数据的数据集，用于训练 K2 学习使用地球科学学术搜索引擎（我们将其命名为 GeoSearch）。然后，我们从 ToolBench 中整合了 8k 条工具指令数据（涉及 arXiv、必应搜索、数据库、天气和 Wolfram Alpha），为工具学习做准备。我们将这个工具指令数据集称为 GeoTool，总共包含 10k 个样本。

- GAKG：GAKG [8] 是一个多模态地球科学学术知识图谱，组织地球科学论文的插图、文本和文献计量数据。每篇论文都有几个地理提及知识公理来连接不同的知识点。

## 4 训练 K2

在本节中，我们建立了一个在特定领域微调大语言模型的方案，并分享了我们在 K2 中采用的设置。

### 4.1 地球科学领域适配方案

由于地球科学是一个相对次要或冷门的研究领域，因此针对此类场景的语言模型很少。然而，先进的自然语言模型和工具可以帮助地球科学家在其研究领域进行数据挖掘和知识发现。因此，学习一个用于知识理解、总结和问答的语言模型是必要的。同时，地球科学拥有丰富的知识积累，如学术论文和科学报告，这为训练大规模语言模型奠定了数据基础。因此，基于地球科学领域的数据，我们探索了一个科学领域适配的方案，并最终获得了 K2。

如图 5 所示，科学领域适配包含三个主要步骤。首先，我们使用领域特定的文本语料库对基础模型进行继续预训练。在本文中，我们使用 LLaMA 作为基础模型。其次，由于指令微调可以使语言模型生成的內容遵循人类指令，我们可以先使用通用指令微调数据（如 Alpaca 和 Natural Instruction）进行指令微调。最后，在学习遵循指令的范式之后，模型可以从重构的领域知识中学习更多信息，我们称之为专业知识指令微调。在消融实验中，我们将进一步验证这一方案的正确性。

### 4.2 继续预训练

在基于地球科学文本语料库进行继续预训练的阶段，我们初始化 LLaMA-7B [47] 检查点，并在 55 亿词元的地球科学语料库上对其进行训练。

LLaMA-7B 的全部参数（67 亿可训练参数）在 4 块 NVIDIA A100-SXM 40GB GPU 上继续预训练一个周期，训练耗时 214 小时。在此阶段，我们设置学习率为 1e-5，全局批次大小为 128，微批次大小为 2。训练的总步数为 30,140 步（其中 1,000 步用于预热）。最后，我们将继续预训练后得到的模型称为 GeoLLaMA，以便更好地区分。

### 第8页

4.3 指令微调

在地球科学文本数据上继续预训练后，我们获得了一个经历了领域迁移的模型。然而，在这个阶段，模型只能完成遵循地球科学知识分布的下一词元生成任务。为了使模型遵从人类指令，我们采用了多任务训练。在此过程中，人类指令呈现多种形式，正如我们的方案所述，包括像 Alpaca 这样的通用任务指令和像 GeoSignal 这样的知识密集型指令。通过实验，我们发现先进行通用指令学习，再进行知识密集型指令学习，有助于提升我们模型的性能，其效果远超混合训练得到的结果。

在指令学习阶段，我们引入了参数高效微调，以帮助我们在低资源环境下完成训练任务。如 [19] 所述，微调过程中的权重更新也具有较低的“内在秩”。因此，根据 LoRA，一个隐藏层 \(h = W_{0}x\)，其中 \(W_{0}\in R^{d*k}\)，修改后的前向传播为：

\[h = W_{0}x + \Delta Wx = W_{0}x + BAx, \quad (1)\]

其中 \(B\in R^{d*r}, A\in R^{r*k}\) 且 \(r << min(d,k)\) 是两个包含可训练参数的低“内在秩”矩阵。此外，在对 LLaMA 进行继续预训练之后，模型对地球科学领域的适应性更加全面。在指令微调阶段，目标是训练模型与人类和专家对齐。我们使用低秩适配来微调模型。

在指令微调中，我们设置学习率为 1e-4，全局批次大小为 128。对于 LoRA 的设置，我们根据实验观察，将 lora_r 设为 8，lora_alpha 设为 16，lora_target_modules 设为 k_proj、q_proj 和 v_proj。使用 LoRA 进行的指令微调仅在一张 NVIDIA GeForce RTX 3090 上训练了 6M 参数，耗时 23 小时。为了使模型表现更好，并在 SFT 阶段注入部分地球科学知识，我们首先使用 Alpaca 指令微调数据训练 GeoLLaMA，我们将其视为人类对齐。然后，我们从该检查点恢复，并使用 GeoSignal 继续微调模型进行进一步训练。我们的实验观察表明，如果混合这些训练数据，性能不会提高。

此外，为了充分利用 K2 的能力，我们在地球科学领域的科学应用中适配了工具学习。正如我们在使用 GeoSignal 微调 GeoLLaMA 以与人类和专家对齐时选择 LoRA 一样，我们在训练 K2 使用工具时也使用了 LoRA。在这个过程中，我们参考了 [38]，并使用我们的工具数据集来学习搜索与地球科学相关的知识和文献。这使得 K2 成为一个可以自主调用外部 API 的实用基础模型。使用 LoRA 进行的工具学习在 8 块 NVIDIA GeForce RTX 3090 上训练了 4M 参数，耗时 21 小时，学习率为 1e-5，最大内容长度为 2,048。

为了更好地理解使模型与人类和专家对齐的方案，我们进行了消融实验来探索细节。我们将由自指令或人工标注构建的通用领域或对话生成数据视为人类对齐数据，并将由特定领域专家标注的数据视为专家对齐数据。如表 7 所示，使用任务特定数据（如 dolly-15k）未能取得良好性能，而使用自指令数据（如 Alpaca-GPT4）的效果也不如使用知识密集型数据。令人惊讶的是，我们发现如果将知识密集型数据 GeoSignal 与人类对齐数据 Alpaca 混合，结果并不理想。最好先使用 Alpaca 使模型遵循人类指令，然后使用 GeoSignal 与专家对齐。

## 6 结论

在本文中，我们介绍了 K2，这是地球科学领域首个大型语言模型和基础模型。K2 能够回答地球科学问题，并凭借其在地球科学领域的专业性遵循地球科学家的指令。我们构建了首个地球科学监督指令数据 GeoSignal。同时，我们建立了地球科学领域首个自然语言处理基准 GeoBench，用于评估模型在地球科学知识理解与应用方面的能力。在我们收集的地球科学基准测试上，K2 与其他类似尺寸的语言模型相比，展现了其专业性和有效性。最后，我们在 https://github.com/davendw49/k2 开源了所有代码、数据和 K2 模型权重。
